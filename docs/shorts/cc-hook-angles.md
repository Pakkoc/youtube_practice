# CC Shorts 훅 타이틀 — 기능 설명형 가이드

CC Shorts는 시리즈 콘텐츠로, 알고리즘 노출보다 **채널 방문 시 탐색**이 핵심 목적.
시청자가 "이건 뭐 하는 기능이지?" → 클릭하는 학습 동선을 만든다.

---

## 역할 분담

| 필드 | 역할 | 설명 |
|------|------|------|
| **line1** | 기능 설명 (큰 네온) | 핵심 기능을 직관적 한국어로 표현 |
| **subDetail** | 명령어 + suffix (손글씨) | `/command · suffix` 패턴 |

---

## line1 작성법

대본에서 **핵심 기능**을 추출하여 직관적 한국어로 표현.

- **권장 7~13자** (네온 폰트에서 임팩트 극대화)
- 기능이 "무엇을 하는지"를 한 마디로 전달
- 전문 용어 최소화, 비개발자도 이해할 수 있는 표현

### 예시

| 대본 핵심 | line1 |
|-----------|-------|
| /init으로 프로젝트 초기 세팅 | 프로젝트 초기화 명령어 |
| /compact로 컨텍스트 압축 | 컨텍스트 압축 기능 |
| /clear로 대화 내역 정리 | 대화 내역 정리 기능 |
| /cost로 토큰 비용 확인 | 토큰 비용 확인 기능 |
| /rewind로 되돌리기 | 작업 되돌리기 명령어 |
| CLAUDE.md로 AI 설정 | AI 맞춤 설정 파일 |
| /context로 참고 자료 추가 | 컨텍스트 참조 기능 |

---

## subDetail 패턴

**`/command · suffix`** 형식. suffix는 아래 풀에서 선택.

### suffix 풀

| suffix | 뉘앙스 |
|--------|--------|
| 기능 살펴보기 | 탐색형 (기본) |
| 한눈에 보기 | 요약형 |
| 완전 정복 | 학습형 |
| 핵심 정리 | 압축형 |
| 바로 써먹기 | 실용형 |

### 예시

| subDetail | 글자 수 |
|-----------|---------|
| /init · 바로 써먹기 | 13 |
| /compact · 핵심 정리 | 14 |
| /clear · 한눈에 보기 | 14 |
| /cost · 기능 살펴보기 | 14 |
| /rewind · 완전 정복 | 13 |
| CLAUDE.md · 핵심 정리 | 15 |

**규칙**: 같은 시리즈 내 연속 동일 suffix 사용 금지. 다양성 유지.

---

## 안티패턴 (금지)

| 유형 | 예시 | 이유 |
|------|------|------|
| **인지 격차 자극형** | "설정 하나로 AI가 달라집니다" | CC Shorts는 탐색 목적 — 호기심 자극 불필요 |
| **불만/고통 토로형** | "AI가 내 코드를 망쳤다" | 호기심 없이 감정만 자극 |
| **고통 공감형** | "AI 때문에 힘들죠" | 톤 반복 + 클릭 동기 부족 |
| **피로감 호소형** | "매번 설명하기 지쳤죠" | 부정적 시작 → 이탈 유발 |
| **범용 클릭베이트** | "99%가 모르는 기능" | 신뢰 하락 |

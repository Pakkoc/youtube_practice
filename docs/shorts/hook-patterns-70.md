# 바이럴 후킹 문구 패턴 70개

쇼츠/릴스 제목 작성 시 참고하는 후킹 패턴 라이브러리.
이 패턴을 **그대로 쓰지 말고**, 대본의 주제에 맞게 **응용**하여 원본 제목을 만든다.

> 출처: SNS 성장프로젝트 탁조아 — "강력한 후킹문구 70개 인스타 릴스 쇼츠 썸네일 글자"

---

## 사용법: 패턴 → line1 변환

이 문서의 패턴을 그대로 쓰지 말고, 대본 주제에 맞게 **구조를 응용**하여 원본 제목을 만든다.

**변환 원칙:**
- `line1`에 메인 타이틀 전체를 넣는다 (최대 16자)
- `line2`는 빈 문자열 `""` (사용하지 않음)
- `subDetail`에 기능명/주제 맥락 보충 (최대 15자)

**변환 예시:**
| 원본 패턴 | line1 | subDetail |
|-----------|-------|-----------|
| "대부분 모르고 지나치는 숨은 진실" | 놓치고 있던 숨은 기능 | Claude Code 필수 설정 |
| "이 실수 하나로 실패할 수 있습니다" | 이 실수면 코드 날아감 | git 복구 명령어 |
| "누구나 따라 할 수 있는 쉬운 방법" | 한 줄이면 세팅 끝 | /init 사용법 |
| "한 번 알면 평생 쓸 수 있는 꿀팁" | 한 번 알면 평생 씀 | 컨텍스트 관리 꿀팁 |

## 교육/테크 콘텐츠 적합도

| 카테고리 | 적합도 | 이유 |
|---------|--------|------|
| **호기심 유발** | ★★★ | 기능 발견, "나도 모르고 있었나?" |

| **강조** | ★★☆ | 필수 기능, 핵심 팁 부각 |
| **간편함** | ★★★ | "이렇게 쉬운 방법이?" 놀라움 |
| **긴급함** | ★★☆ | 신기능, 업데이트, 트렌드 변화 |
| 권위 | ★☆☆ | 과장되기 쉬움, 공식 문서 인용 시만 |
| 신뢰 | ★☆☆ | 수치 기반 증거가 있을 때만 |

**교육/테크 쇼츠에서 가장 많이 쓸 카테고리: 1(호기심), 2(경고), 4(강조), 5(간편함).**

## CC Shorts 전용 앵글 (인지 격차 자극형)

범용 70개 패턴과 별개로, CC Shorts는 **"인지 격차 자극형"** 앵글 6가지를 사용한다.
기존 고통 공감형("AI 때문에 힘들다")에서 벗어나, 시청자가 모르는 사실/오해/숨겨진 원리로 호기심을 자극.

| # | 앵글 | 공식 | line1 예시 | 적합한 소재 |
|---|------|------|-----------|------------|
| 1 | **결과 대비형** | "X 하나로 Y가 달라집니다" | 설정 하나로 AI가 달라집니다 | 초기 설정, 한 줄 명령어 |
| 2 | **원인 밝히기형** | "X의 진짜 이유" | AI가 멍청해지는 진짜 이유 | AI 성능 저하, 반복 실수 |

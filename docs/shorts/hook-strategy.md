# Shorts Hook & Script Strategy Reference

쇼츠 대본 재구성 시 참조하는 레퍼런스. `/generate-shorts-script` 스킬에서 로드.

## 1. 콘텐츠 유형 분류 (5가지)

대본을 분석하여 아래 5가지 유형 중 하나로 분류. 복합 유형이면 **주요 유형 1개 + 보조 유형 1개**까지.

| 유형 | 설명 | 특징 | 예시 |
|------|------|------|------|
| **정보/뉴스** | 새로운 사실, 트렌드, 발견 전달 | 구체적 팩트, 수치, 출처 | "GPT-5 발표", "애플 신제품" |
| **하우투/팁** | 방법론, 실용 정보 제공 | 단계별 설명, 실행 가능한 조언 | "N8N 자동화 설정법", "프롬프트 작성법" |
| **리뷰/비교** | 제품/서비스/아이디어 평가 | 장단점, A vs B, 순위 | "Claude vs GPT 비교", "TOP 3 AI 도구" |
| **의견/경험** | 개인 관점, 경험담, 주장 | 감정, 스토리, 반전 | "AI로 월 500만원 번 후기", "퇴사 후 깨달은 것" |
| **데모/쇼케이스** | 실제 동작, 결과물 시연 | 비포/애프터, 라이브 프로세스 | "실시간 자동화 시연", "AI가 만든 영상" |

## 2. 유형별 추천 훅 매핑

### 정보/뉴스형

| 훅 패턴 | 공식 | 예시 |
|---------|------|------|
| **"혹시 ~에 대해 들어보셨나요?"** | FOMO + 호기심 | "GPT-5의 숨겨진 기능, 들어보셨나요?" |
| **"~라는 사실, 알고 계셨나요?"** | 놀라움 + 정보 격차 | "Claude가 코드를 직접 실행한다는 사실, 알고 계셨나요?" |
| **"저도 이걸 이제야 알았습니다"** | 공감 + 겸손 | "저도 오늘 처음 안 건데요, AI가 이미지를 이렇게 만들 수 있었습니다" |

**핵심 심리**: 호기심의 간극(Curiosity gap) — 모르면 손해인 느낌

### 하우투/팁형

| 훅 패턴 | 공식 | 예시 |
|---------|------|------|
| **"~하기 전 꼭 하는 3가지"** | 구체적 숫자 + 실용 | "N8N 워크플로 만들기 전 꼭 하는 3가지" |
| **"[목표]를 달성하려면 이것을 시도해 보세요"** | 목표 제시 + 솔루션 | "유튜브 자동화하려면 이 도구 하나만 쓰세요" |
| **"~ 만드는 법"** | 직접적 가치 약속 | "AI로 30초만에 자막 만드는 법" |
| **"~를 위한 간단한 꿀팁"** | 지름길 약속 | "프롬프트 비용 절반으로 줄이는 꿀팁" |

**핵심 심리**: 가치 약속 — 이 영상을 보면 구체적으로 무엇을 얻는지 즉시 파악

### 리뷰/비교형

| 훅 패턴 | 공식 | 예시 |
|---------|------|------|
| **"X vs Y"** | 대립 구도 + 경쟁심 | "Claude vs GPT-5, 진짜 승자는?" |
| **"제 인생 최고의 제품입니다"** | 극단적 추천 | "올해 써본 AI 도구 중 이게 최고입니다" |
| **"가장 좋은 ~ TOP 3"** | 리스트 + 순위 | "2026년 AI 코딩 도구 TOP 3" |

**핵심 심리**: 비교 + 사회적 증거 — 어느 쪽이 더 나은지 알고 싶은 본능

### 의견/경험형

| 훅 패턴 | 공식 | 예시 |
|---------|------|------|
| **"저는 절대 이것을 하지 않습니다"** | 반대 의견 + 차별화 | "저는 절대 GPT-4를 안 씁니다. 그 이유는..." |
| **"사람들이 알려주길 원치 않는 비밀"** | 음모 + 희소성 | "AI 업계가 숨기고 싶어하는 진짜 비용" |
| **"여러분이 고생하지 않게 대신 해봤습니다"** | 영웅 서사 + 대리 경험 | "AI 도구 10개 전부 써봤습니다. 여러분은 이것만 쓰세요" |

**핵심 심리**: 패턴 파괴(Pattern interruption) — 예상과 다른 의견이 시선을 붙잡음

### 데모/쇼케이스형

| 훅 패턴 | 공식 | 예시 |
|---------|------|------|
| **"이런 건 태어나서 처음 보실걸요"** | 도전 + 놀라움 | "AI가 영상을 이렇게 만드는 건 처음 보실걸요" |
| **"구글에 직접 검색해 봤습니다"** | 친숙함 + 실험 | "AI 자동화 진짜 되는지 직접 해봤습니다" |
| **"인생 최고의 ~를 경험하는 법"** | 극단적 약속 | "인생 최고의 프레젠테이션을 AI로 만드는 법" |

**핵심 심리**: 시각적 증거 — 직접 보여주는 것이 가장 강력한 훅

## 3. 쇼츠 대본 구조 공식

### 3.1 전체 구조 (45-60초, 4-6문단)

```
[문단 1] 훅 (3초, 15-25자)
  ↓ 즉시 핵심으로
[문단 2] 핵심 포인트 1 (10-15초, 40-70자)
  ↓
[문단 3] 핵심 포인트 2 또는 구체적 예시 (10-15초, 40-70자)
  ↓
[문단 4] 결론/반전/CTA (5-10초, 25-50자)
```

- 총 4문단이 기본, 내용이 많으면 5-6문단까지 확장
- **절대 6문단 초과 금지** (60초 제한)

### 3.2 훅 문단 작성법 (문단 1)

| 규칙 | 설명 |
|------|------|
| **15-25자** | TTS 기준 3초 이내 |
| **한 문장** | 두 번째 문장이 필요하면 너무 복잡 |
| **구체적 약속** | 결과물, 실수, 반전을 명시 |
| **즉각적 맥락** | 누구를 위한 영상인지 즉시 파악 |

**훅 작성 공식:**
- `[상황/대상] + [놀라운 사실/반전]` → "코딩 없이 월 500만원 벌 수 있다면요?"
- `[문제] + [해결 암시]` → "유튜브 편집에 3시간 쓰고 계세요?"
- `[숫자] + [구체적 결과]` → "30초만에 자막이 완성됩니다"

### 3.3 시청자 공감 기반 훅 작성법

패턴은 도구이고, 공감이 본질이다. 70개 훅 패턴을 적용하기 **전에** 시청자가 누구이고 무엇을 원하는지 먼저 파악해야 훅이 "정보 전달"이 아닌 "감정 연결"이 된다.

#### 왜 중요한가

같은 주제라도 타겟 시청자에 따라 훅이 달라진다:
- 주제: "RAG 기술 동향"
- 초보자 타겟: "AI 챗봇이 거짓말하는 이유, 알고 계셨나요?" (두려움 + 호기심)
- 현업 엔지니어 타겟: "RAG는 죽었다. 요즘 이 말 많이 보셨죠?" (불안 공감 + 인지 격차)

#### 공감 분석 3단계

| 단계 | 질문 | 출력 |
|------|------|------|
| **1. 페르소나** | 이 주제를 검색/클릭할 사람은? (역할, 상황, 고민) | "RAG 프로젝트 경험있는 AI 엔지니어, 최근 불안감" |
| **2. 숨은 욕구** | 듣고 싶은 말 / 두려운 것 / 궁금한 것 | "내 경험이 헛되지 않았다 / 방향 잘못 잡은 거 아닌가 / 앞으로 어떻게?" |
| **3. 공감 워딩** | "맞아 맞아!" 반응을 유발할 표현 3개 | "죽은 건 RAG가 아닙니다", "방향만 바꾸면 됩니다" |

#### 훅에 공감 워딩 녹이는 법

1. 공감 워딩 중 가장 강한 것을 훅 문단에 직접 사용하거나 변형
2. 나머지는 본문 첫 1-2문단에 배치 (시청자가 "이 사람 내 상황 아는구나" 느끼도록)
3. 훅 패턴(2절 참조)은 공감 워딩을 **포장하는 형식**으로 활용

**좋은 예 (공감 → 패턴):**
```
공감 워딩: "내가 해온 RAG 작업이 헛되지 않았다"
패턴: 통념 뒤집기
→ 훅: "RAG는 죽었다. 요즘 이 말 많이 보셨죠? 근데 죽은 건 RAG가 아닙니다."
```

**나쁜 예 (패턴만, 공감 없음):**
```
패턴: 정보/뉴스형 — "~라는 사실, 알고 계셨나요?"
→ 훅: "RAG 기술이 변하고 있다는 사실, 알고 계셨나요?"
(누구에게 말하는 건지, 왜 봐야 하는지 감정적 연결 없음)
```

### 3.4 본문 압축 원칙

원본 대본이 길 경우 다음 원칙으로 압축:

1. **핵심 1-2개만 선택**: 원본의 모든 포인트를 넣으려 하지 말 것
2. **배경 설명 제거**: "~의 역사를 보면", "~란 무엇인가 하면" 등 도입부 삭제
3. **결론 먼저**: 결론을 앞에 놓고 근거는 뒤에 (역피라미드)
4. **구체적 수치 유지**: 추상적 설명보다 "30%", "3배", "100만" 등 숫자가 강력
5. **하나의 예시만**: 여러 예시 중 가장 인상적인 것 하나만 선택

### 3.5 CTA/마무리 패턴

| 패턴 | 예시 | 적합한 유형 |
|------|------|------------|
| **반전 마무리** | "근데 진짜 무서운 건 따로 있습니다" | 정보/뉴스, 의견 |
| **행동 유도** | "지금 바로 해보세요" | 하우투/팁 |
| **질문 마무리** | "여러분은 어떻게 생각하세요?" | 리뷰/비교, 의견 |
| **요약 강화** | "이 한 가지만 기억하세요" | 전체 유형 |
| **다음 편 암시** | "다음 편에서 더 자세히 알려드릴게요" | 시리즈 |

## 4. 훅 타이틀 작성 규칙

화면 상단에 표시되는 3-layer 텍스트. 대본의 훅 문단과는 별개.

### 형식
- **line1** (민트/강조색): 메인 타이틀 전체 — **항상 1줄 렌더링** (`whiteSpace: "nowrap"` + 동적 폰트 스케일링, 줄바꿈 절대 없음)
- **line2**: 빈 문자열 `""` (사용하지 않음)
- **subDetail** (손글씨): 최대 **15자**, 기능/주제 맥락
- 권장 10~13자 (임팩트↑), 최대 ~20자 (폰트 자동 축소). 8자 이하=~200px, 13자=~110px, 20자=~70px

### 핵심 원칙
- **line1 = 인지 격차 자극** (시청자가 모르는 사실/오해/숨겨진 원리), **subDetail = 맥락** (이 영상이 뭔지)
- 기능명을 나열하는 것은 라벨이지 제목이 아님
- **`docs/shorts/hook-patterns-70.md`** 참조하여 70가지 패턴의 구조를 응용

### CC Shorts 전용 6가지 앵글 카테고리

대본에서 시청자가 모르는 사실/오해/숨겨진 원리를 추출하여 아래 앵글 중 하나로 line1을 작성.
상세 가이드: `docs/shorts/cc-hook-angles.md`

| # | 앵글 | 공식 | line1 예시 |
|---|------|------|-----------|
| 1 | **결과 대비형** | "X 하나로 Y가 달라집니다" | 설정 하나로 AI가 달라집니다 |
| 2 | **원인 밝히기형** | "X의 진짜 이유" | AI가 멍청해지는 진짜 이유 |
| 3 | **숨겨진 원리형** | "N개 X가 다 같은 Y" | 18개 AI 모델이 다 같은 문제 |
| 4 | **숫자 충격형** | "X, N배/N% Y" | AI 코드, 버그 1.7배 많습니다 |
| 5 | **잘못된 습관 지적형** | "X하면 잘못 쓰는 겁니다" | Ctrl+Z로 되돌리면 잘못입니다 |
| 6 | **통념 뒤집기형** | "X은 Y이 아닙니다" | 같은 실수 반복, AI 탓 아닙니다 |

### CC Shorts subDetail 패턴

`/명령어 기능 살펴보기` 고정 패턴. 예: `/init 기능 살펴보기`, `/compact 기능 살펴보기`

### 톤 다양성 규칙

- 3개 후보 생성 시 반드시 **서로 다른 앵글**에서 생성
- 같은 시리즈 내 동일 앵글 연속 사용 금지

### 안티패턴 (금지)
- 기능명 나열: 기능명 + 설명 = 후킹이 아님
- 패턴 복붙: 70개 패턴은 참고용이지 그대로 쓰면 안 됨
- 범용 클릭베이트: "99%가 모르는", "충격 실화"
- 내용과 무관한 과장: 영상에서 다루지 않는 내용
- 너무 추상적: "놀라운 사실", "대박" (구체성 없음)
- 13자 이상: 폰트가 너무 작아져 가독성 저하
- **불만/고통 토로형 금지**: "AI가 X했다", "X하기 지쳤죠" 등 감정만 자극하는 패턴

## 5. 심리 트리거 체크리스트

훅 작성 후 아래 5가지 중 최소 1개를 자극하는지 확인:

- [ ] **호기심**: 빠진 정보를 알고 싶게 만드는가?
- [ ] **공감**: "나도 그래!" 또는 "맞아 맞아" 반응을 유발하는가?
- [ ] **불안/FOMO**: 모르면 손해, 놓치면 안 될 것 같은 느낌인가?
- [ ] **놀라움**: 예상을 깨는 정보나 반전이 있는가?
- [ ] **인지 격차**: 시청자가 모르는 사실/오해/숨겨진 원리를 건드리는가? (CC Shorts 핵심)

2개 이상 자극하면 바이럴 가능성 높음.

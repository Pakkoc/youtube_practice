# 쇼츠 캡션 작성 가이드라인

## 브랜드 톤 & 보이스

### 핵심 원칙
- **1인칭 경험 기반**: 직접 해본 경험에서 나오는 자연스러운 권위. "~해봤는데", "~를 진행했었습니다"
- **솔직한 진정성**: 어려움도 숨기지 않음. "쉽지 않다는 것은 익히 들었습니다만", "온몸이 아프지만"
- **겸손 + 전문성**: 자랑이 아닌 사실 기반 전문성. "대기업과 여러 대표님 대상으로", "컨설팅 받은 내용을 실제로 구현"
- **독자 가치 중심**: 항상 독자가 얻을 수 있는 것을 강조
- **구체적 숫자/데이터**: 추상적 표현 대신 실제 수치. "20분", "2분 만에", "8시간 마스터클래스"

### 문체
- **~습니다 경어체** 기반, 딱딱하지 않게
- **구어체와 문어체 적절히 혼합**: "~거든요", "~인데요" 자연스럽게 사용
- **이모지 최소**: :) 정도만 허용. 이모지 남발 금지
- **한국어 + 영어 기술 용어 혼용**: n8n, Claude Code, MCP 등 기술 용어는 영어 그대로
- **문단 간 빈 줄**로 호흡 조절

### 금지 사항
- ❌ 과장된 클릭베이트 ("충격!", "미쳤다!", "꼭 봐야 할")
- ❌ 이모지 과다 사용
- ❌ 빈 수식어 ("엄청난", "대단한", "놀라운")
- ❌ 명령형 CTA ("지금 당장 구독하세요!")
- ❌ 해시태그 사용 금지

## 캡션 구조 (Threads/인스타/유튜브 쇼츠)

### 기본 구조 (3~6줄)
```
[경험 기반 도입 또는 상황 설정] — 1~2문장
[핵심 가치/인사이트] — 1~2문장
[구체적 맥락 또는 뒷이야기] — 1~2문장 (선택)
[자연스러운 CTA] — 1문장 (선택)
```

### 도입부 패턴
- 경험 기반: "~를 N개월 써보고 느낀 건데요"
- 문제 제기: "보통 ~하면 뇌정지 오신 경험 있으실 겁니다"
- 결과 먼저: "이 워크플로우 만드는 데 20분도 안 걸렸습니다"
- 비교/대조: "대부분 ~하지만, 저는 ~"

### CTA 패턴 (자연스러운 것만)
- "관심 있으신 분들은 풀영상 참고해보시면 좋겠습니다"
- "자세한 방법은 영상으로 만들어놨습니다"
- "댓글에 관련 링크 남겨놓겠습니다"
- CTA 없이 인사이트만 전달해도 OK

## 쇼츠별 캡션 특화

- **데모형 쇼츠**: 결과를 먼저 보여주고 → 어떻게 만들었는지 한 줄 → 풀영상 CTA
- **팁/노하우형**: 문제 상황 → 해결 인사이트 → 경험 기반 근거
- **비교형**: 기존 방식 단점 → 대안 제시 → 왜 이게 더 나은지

# Hook Titles (훅 타이틀 생성 규칙)

## 구조

HookTitle 컴포넌트는 **항상 1줄 렌더링** (`whiteSpace: "nowrap"` + 동적 폰트 스케일링).
글자 수가 많으면 폰트가 자동으로 줄어들고, 적으면 커진다. 줄바꿈은 절대 발생하지 않는다.

| 필드 | 표시 | 제한 |
|------|------|------|
| **line1** | 민트색 메인 타이틀 (1줄 고정, 동적 폰트) | 권장 10~13자 (임팩트), 최대 ~20자 (폰트 작아짐) |
| **line2** | 사용하지 않음 | 항상 빈 문자열 `""` |
| **subDetail** | 손글씨 부제 | 최대 15자 |

### 폰트 스케일 참고 (1080px 기준)

- 8자 이하: ~200px (최대, 화면 가득)
- 10자: ~150px (매우 큼)
- 13자: ~110px (이상적)
- 16자: ~90px (읽기 좋음)
- 20자: ~70px (최소 권장)

## 역할 분담

- **CC Shorts(cc-\*)**: line1 = 기능 설명 (핵심 기능을 직관적 한국어로), subDetail = `/command · suffix`
- **일반 프로젝트**: line1 = 인지 격차 자극, subDetail = 맥락 (기존 방식 유지)

## 만드는 법

1. **CC Shorts(cc-\*)**: 대본에서 **핵심 기능**을 추출하여 직관적 한국어로 line1 작성 (상세: `docs/shorts/cc-hook-angles.md`)
   - 권장 7~13자, 기능이 "무엇을 하는지"를 한 마디로 전달
   - 예: "프로젝트 초기화 명령어", "컨텍스트 압축 기능", "대화 내역 정리 기능"
2. **일반 프로젝트**: `docs/shorts/hook-patterns-70.md`에서 적합한 카테고리 선택 (호기심/경고/강조/간편함 추천)
3. 패턴의 **구조**를 응용하여 원본 제목 생성 (복사 금지)

## CC Shorts subDetail 패턴

CC Shorts의 subDetail은 **`/command · suffix`** 패턴.
suffix 풀: `기능 살펴보기` / `한눈에 보기` / `완전 정복` / `핵심 정리` / `바로 써먹기`
예: `/init · 바로 써먹기`, `/compact · 핵심 정리`, `/clear · 한눈에 보기`

## 피할 것

- 기능명 나열 (라벨 =/= 제목)
- 패턴 복붙
- 합산 17자 이상
- "충격!", "99%가 모르는" 등 뻔한 클릭베이트
- **불만/고통 토로형 금지**: "AI가 X했다", "X하기 지쳤죠" — 감정만 자극, 클릭 동기 부족
- **인지 격차 자극형 금지** (CC Shorts 한정): "설정 하나로 AI가 달라집니다" — CC는 탐색 목적, 호기심 자극 불필요

## 후보 3개 -> 선택

3개 후보는 반드시 서로 다른 앵글/카테고리에서 생성. 동일 앵글 중복 금지.
각 후보에 앵글명을 표기하여 다양성을 시각적으로 확인. 사용자에게 보여주고 최선 선택.

## 저장

`projects/$PROJECT/shorts_slides/slides/hook_titles.json` — 모든 엔트리 동일 제목.

`line1`에 메인 타이틀 전체, `line2`는 항상 빈 문자열 `""`.

```json
[
  {"index": 1, "line1": "프로젝트 초기화 명령어", "line2": "", "subDetail": "/init · 바로 써먹기"},
  {"index": 2, "line1": "프로젝트 초기화 명령어", "line2": "", "subDetail": "/init · 바로 써먹기"},
  ...
]
```

# Carousel Copywriting — 프리프로덕션 카피 설계

주제/아이디어를 받아 인스타그램 캐러셀 카드별 텍스트를 설계하고, 8가지 평가지표로 자가평가한 뒤 개선까지 수행한다.

**이 스킬이 해결하는 문제**: `/generate-carousel`은 비주얼 프로덕션에 집중한다. 하지만 아무리 디자인이 좋아도 텍스트가 약하면 스크롤은 멈추지 않는다. 이 스킬은 비주얼 이전 단계에서 텍스트의 밀도, 맥락, 설득력을 확보한다.

**인풋**: 주제/아이디어, 기존 script.txt, 또는 자유 형식의 콘텐츠 소스
**아웃풋**: `projects/$PROJECT/carousel/copy_deck.md`

---

## Step 1: 소스 분석 & 오디언스 매핑

### 1a. 소스 읽기

```
projects/$PROJECT/script.txt            -- 있으면 읽기
projects/$PROJECT/carousel/copy_deck.md -- 기존 카피 있으면 → 개선 모드
```

소스가 없으면 사용자에게 주제/아이디어를 질문한다.

### 1b. 핵심 추출

| 항목 | 내용 |
|------|------|
| **원 메시지** | 이 캐러셀이 전달하려는 핵심 한 문장 |
| **타겟 독자** | 누가 읽는가 (직업, 관심사, 지식 수준) |
| **독자의 현재 고통** | 지금 뭐가 불편한가 |
| **약속하는 변화** | 다 보면 뭐가 달라지는가 |
| **증거 유형** | 데이터 / 사례 / 비교 / 권위 중 무엇으로 설득하는가 |

### 1c. 브랜딩 + 배경 이미지 설정 확인

```
Read prompts/channel_identity.txt
Read config/config.base.yaml → carousel.branding 섹션
Read config/config.base.yaml → carousel.background.enabled
```

`brand_voice`와 `target_audience`를 카피 톤 기준으로 삼는다.

**AI 배경 이미지 모드**: `carousel.background.enabled`가 `true`이면 Step 4에서 선택적 카드에 `image_prompt` 필드를 작성한다. `false`(기본값)이면 `image_prompt` 작성을 건너뛴다 — 순수 TSX로만 디자인.

## Step 2: 훅 엔지니어링 (Cover Copy)

인스타그램 피드에서 첫 장이 3초 안에 스크롤을 멈추지 못하면, 나머지 카드는 존재하지 않는 것과 같다.

### 2a. 훅 공식: "문제 + 궁금증 + 숫자"

| 요소 | 역할 | 예시 |
|------|------|------|
| 문제 | 독자의 고통을 찌름 | "캐러셀 조회수 망치는" |
| 궁금증 | 답이 궁금하게 | "치명적 실수" |
| 숫자 | 구체성 → 신뢰 | "5가지" |

### 2b. 훅 후보 3개 생성

각 후보에 대해:
- **제목** (10자 이내 목표, 최대 15자)
- **부제** (제목 보완, 제목만으로 클릭 동기 충분해야 함)
- **적용 패턴**: 숫자+결과 | 반전/도발 | 질문형 | 비교/대비

### 2c. 훅 선정

| 기준 | 가중치 |
|------|--------|
| 3초 내 스크롤 멈춤 가능성 | ★★★ |
| 문제+궁금증+숫자 공식 부합 | ★★★ |
| 10자 이내 달성 | ★★ |
| 떡밥 회수 가능한 키워드 포함 | ★★ |
| 타겟 독자 공감 | ★★ |

3개 후보 중 최고를 선택하고, **선택 이유를 명시**.

### 2d. 커버 키워드 확정

CTA에서 회수할 떡밥 키워드를 확정한다.
예: 커버 "자동화의 비밀" → CTA "이제 자동화를 시작할 차례"

## Step 3: 세컨드 슬라이드 전략

두 번째 장은 첫 번째만큼 중요하다. 인스타그램은 사용자가 첫 번째만 보고 넘기면, **두 번째 장을 메인으로 재노출**한다. 세컨드 슬라이드는 사실상 두 번째 커버다.

### 3가지 검증된 세컨드 전략

| 유형 | 설명 | 적합한 콘텐츠 |
|------|------|-------------|
| **미리보기형** | 이어질 내용을 미리 보여 기대감 형성 | 리스트형, 단계형 |
| **오해/자칭형** | 빈문을 던져 호기심 자극 | 오해 바로잡기, 반전 |
| **한판정리형** | 전체 내용을 한눈에 요약 | 비교, 종합 정리 |

선택한 유형과 그 이유를 명시.

## Step 4: 카드별 카피 작성

### 4a. 서사 구조 설계

```
공감 (카드 2~3) → 전환 (카드 3~4) → 증거 (카드 5~7) → 실천 (카드 N-1~N)
```

카드 수: **8~12장**. 꽉 찬 7장이 허전한 10장보다 낫다.

### 4b. 카드 테이블

| # | 역할 | 감정 단계 | 온도 | 핵심 메시지 (1문장) | 본문 (150자 이내) | 스와이프 연결 |
|---|------|----------|------|-------------------|------------------|-------------|
| 1 | 커버 | 호기심 | ●●●●○ | [훅 제목] | [부제] | → 문제 제기 |
| 2 | 세컨드 | [유형] | ●●○○○ | ... | ... | → ... |
| ... | | | | | | |
| N | CTA | 여운+행동 | ●●●●● | [떡밥 회수] | [행동 촉구] | — |

### 4c. 카드별 작성 원칙

**한 장 = 하나의 메시지**
- 미니 제목 + 이를 설명하는 본문
- 본문 150자 이내 (인스타 사용자 평균 소비 시간 3~5초)
- 설명이 길어지면 2장으로 나눌 것

**AI 안티패턴 + 언어 경제성**: 구체 숫자/대비/개조식 사용, AI스러운 표현 즉시 제거.
-> 상세 기준: `remotion/docs/carousel-editorial-strategy.md` § 5-6

**스와이프 연결**: 내용 자체로 자연스럽게 연결 (질문→답, 문제→해결, 전제→결론).
-> 상세: `remotion/docs/carousel-editorial-strategy.md` § 3

### 4d. AI 배경 이미지 프롬프트 (`carousel.background.enabled: true`일 때만)

`carousel.background.enabled: false`이면 건너뛴다. `true`이면 선택적 카드(3~5장)에 `image_prompt` 필드를 추가한다.
-> 카드 선택 기준 + 프롬프트 작성법: `remotion/docs/carousel-ai-backgrounds.md`

## Step 5: 자가평가 (8가지 메트릭)

작성 완료 후 아래 8가지 메트릭으로 자가평가한다. 각 5점 만점, 총 40점.
상세 채점 기준과 카드별 예시: `remotion/docs/carousel-copywriting-rubric.md`

### 평가 기준 요약

| # | 메트릭 | 5점 (우수) | 3점 (보통) | 1점 (미흡) |
|---|--------|----------|----------|----------|
| 1 | **훅 임팩트** | 문제+궁금증+숫자 완벽, ≤10자 | 공식 부분 적용, ≤15자 | 공식 미적용, 장황 |
| 2 | **세컨드 전략** | 3유형 중 명확 선택, 스와이프 동기 강력 | 유형 선택했으나 실행 미흡 | 전략 없음 |
| 3 | **정보 밀도** | 카드당 1메시지, ≤150자 | 대부분 준수, 1-2장 초과 | 과부하, 다중 메시지 |
| 4 | **언어 경제성** | 완전 개조식, 숫자 활용 | 대부분 개조식, 일부 서술체 | 장황한 서술체 |
| 5 | **감정 곡선** | 공감→전환→증거→실천 완벽, 온도변화 2회+ | 흐름 있으나 단조 | 감정 곡선 없음 |
| 6 | **스와이프 동기** | 매 카드 끝에서 다음 궁금, 자연 Bridge | 일부 카드만 연결 | 카드간 단절 |
| 7 | **떡밥 회수** | 커버 키워드 CTA에서 완벽 회수 + 완결감 | 부분 회수 | 미회수 |
| 8 | **일관성** | 텍스트 분량/구조 일관 | 대부분 일관 | 카드마다 들쭉날쭉 |

### 등급 기준

| 총점 | 등급 | 판정 |
|------|------|------|
| 35-40 | **A** | 퍼블리시 레디 → 바로 `/generate-carousel` 진행 |
| 28-34 | **B** | 소폭 수정 후 사용 가능 |
| 20-27 | **C** | 주요 수정 필요 |
| ≤19 | **D** | 재작성 권장 |

### 자가평가 수행

카드 테이블 완성 후 **반드시** 다음을 수행:

1. 8가지 메트릭 각각에 대해 점수(1-5)와 근거를 기록
2. 총점과 등급 산출
3. **3점 이하 메트릭**과 해당 카드를 특정
4. B등급 이하이면 Step 6으로 진행

---

## Step 6: 개선 (B등급 이하 시)

### 6a. 약점 식별

3점 이하 메트릭과 해당 카드를 특정한다.

### 6b. 메트릭별 개선 전략

| 약점 메트릭 | 개선 방향 |
|------------|----------|
| 훅 임팩트 ≤3 | 다른 훅 패턴으로 재작성 (예: 반전→숫자+결과) |
| 세컨드 전략 ≤3 | 3유형 중 다른 유형으로 교체 |
| 정보 밀도 ≤3 | 과부하 카드 분할, 또는 병합으로 밀도 조절 |
| 언어 경제성 ≤3 | 문장→개조식 전환, 숫자 대체, 조사 제거 |
| 감정 곡선 ≤3 | 온도 배치 재조정, 전환점 추가 |
| 스와이프 동기 ≤3 | 카드 연결고리 추가 (질문→답, 문제→해결) |
| 떡밥 회수 ≤3 | 커버 키워드를 CTA에 명시적으로 반영 |
| 일관성 ≤3 | 텍스트 분량 ±20% 내로 통일, 구조 패턴 통일 |

### 6c. 수정 → 재평가 루프

1. 약점 카드의 카피를 수정
2. Step 5 자가평가 재실행
3. **A등급(35+) 달성까지 반복** (최대 3회)
4. 3회 후에도 B라면 사용자에게 판단 위임

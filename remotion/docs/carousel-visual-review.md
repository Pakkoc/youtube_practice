# Carousel Visual Review Guide

렌더링 완료 후 시각 검수 프로세스. `/generate-carousel` Step 7에서 참조.

---

## 1. 자동 코드 검증

```bash
uv run python scripts/validate_carousel.py $PROJECT
```

- CRITICAL 에러 → 코드 수정 먼저 (시각 검수 미진행)
- WARNING만 → 주의 표시 후 시각 검수 진행
- 모두 PASS → 카드별 검수로

---

## 2. 카드별 시각 검수

각 PNG를 Read 도구로 열어 직접 검사. 카드당 6항목:

1. **이미지 가시성**: 피사체 보이는가, cover 크롭으로 잘림 없는가
2. **텍스트 가독성**: 대비, 오버플로, 폰트 크기, 한국어 렌더링
3. **레이아웃 밸런스**: 빈 공간, 밀집도, 정렬, PageDots 위치
4. **카드간 일관성**: 폰트 크기 통일, 색상 패턴, 마진 통일 (종합 판정)
5. **정보 전달력**: 3초 내 핵심 파악, 시각 계층, 데이터 정확성
6. **서사 흐름**: 공감→전환→증거→실천 흐름, Bridge 연결, 온도 곡선

---

## 3. 검수 리포트 템플릿

```
## 캐러셀 검수 리포트: $PROJECT

### 종합 판정: PASS / FAIL (N개 수정 필요)

| 카드 | 이미지 | 가독성 | 레이아웃 | 일관성 | 전달력 | 서사흐름 | 판정 |
|------|--------|--------|----------|--------|--------|----------|------|

### FAIL 상세
(구체적 문제 + TSX 코드 레벨 수정안)
```

---

## 4. 수정 → 재렌더링 → 재검수 루프

1. FAIL 카드에 구체적 수정안 제시
2. 사용자 승인 후 Edit으로 TSX 수정
3. 수정 카드만 재렌더링:
   ```bash
   uv run python scripts/regenerate_carousel.py $PROJECT <card_numbers> --force
   ```
4. 해당 카드만 재검수
5. 모든 카드 PASS까지 반복

---

## 5. 최종 승인

모든 PASS 시 전체 PNG 경로 리스트 제공 + 사용자 최종 확인.

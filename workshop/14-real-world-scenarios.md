# 실전 시나리오 가이드

> 채널 유형별 설정 + 대본 패턴 + 운영 전략 완전 워크스루

> 난이도: 심화 | 소요: 읽기 20분

---

## Part 1: 시나리오를 선택하세요

이 문서는 4가지 채널 유형에 대한 완전한 실전 가이드입니다. 각 시나리오에는 컨셉 정의부터 소재 발굴, 대본 패턴, 추천 설정값, 스킬 활용 예시, 운영 루틴까지 포함되어 있습니다.

자신의 채널과 가장 비슷한 시나리오를 골라 따라하세요. 정확히 맞는 것이 없다면 가장 가까운 것을 선택하고 Part 6에서 나만의 시나리오로 커스터마이징합니다.

| 시나리오 | 채널 유형 | 주력 포맷 | 업로드 빈도 |
|----------|----------|----------|------------|
| A | IT/기술 뉴스 | 영상 + 쇼츠 | 주 3~5회 |
| B | 교육/튜토리얼 | 영상 + 캐러셀 | 주 2회 |
| C | 트렌드/바이럴 | 쇼츠 중심 | 매일 1~2개 |
| D | AI 활용 사례 | 캐러셀 + 쇼츠 | 주 3회 |

```
어떤 시나리오가 나와 맞을까?

질량 중심 콘텐츠?
├── 뉴스/속보 ─────────→ 시나리오 A
├── 강의/튜토리얼 ─────→ 시나리오 B
├── 바이럴/트렌드 ─────→ 시나리오 C
└── 사용법/활용 사례 ──→ 시나리오 D
```

> **핵심:** 4가지 시나리오 중 하나를 골라 끝까지 따라하면 채널 세팅부터 운영 루틴까지 한번에 잡힙니다. 나중에 Part 6에서 자기만의 버전으로 조정하세요.

## Part 2: 시나리오 A -- IT/기술 뉴스 채널

### 컨셉

AI, 테크 뉴스를 빠르게 영상화하여 전달하는 채널입니다. 속도가 핵심 경쟁력이므로 소재 발굴부터 업로드까지 최대한 자동화합니다.

### 소재 발굴

```
Claude Code에게:
"최근 1주일간 AI 분야 주요 뉴스 5개를 조사해줘.
각 뉴스의 핵심과 왜 지금 다루면 좋은지 알려줘."
```

- `/detect-releases`로 Claude Code 신규 릴리즈 감지 -- CC 업데이트 뉴스의 자동 소재화
- X(트위터), Reddit, Hacker News 트렌드 활용
- 제품 출시일 캘린더를 미리 파악해두면 "예정된 뉴스"로 선제 대응 가능

### 대본 패턴

```
[인트로 - 속보 톤]
오늘 OpenAI에서 GPT-5를 공식 발표했습니다.
이번 업데이트의 핵심 변화 3가지를 빠르게 정리해 드리겠습니다.

[포인트 1 - 구체적 사실]
첫 번째, 컨텍스트 윈도우가 2백만 토큰으로 확장되었습니다.
이전 모델 대비 4배입니다.

[포인트 2]
두 번째, 멀티모달 입출력이 네이티브로 통합되었습니다.
이미지, 오디오, 영상을 하나의 프롬프트에서 처리할 수 있습니다.

[포인트 3]
세 번째, API 가격이 50% 인하되었습니다.
개발자들에게는 가장 반가운 소식일 겁니다.

[아웃트로 - CTA]
이런 AI 뉴스를 빠르게 받아보고 싶다면 구독과 알림 설정 부탁드립니다.
```

Claude Code 프롬프트:

```
"GPT-5 발표" 주제로 3분짜리 IT 뉴스 대본을 작성해줘.
속보 톤으로, 핵심 포인트 3개를 빠르게 정리하는 구조로.
projects/news-001/script.txt에 저장해줘.
```

### 추천 설정

```yaml
# TTS: 빠르고 안정적인 뉴스 톤
tts:
  elevenlabs:
    speed: 1.15              # 빠른 정보 전달
    stability: 0.85          # 안정적이고 신뢰감 있는 톤

# B-roll: 테크 느낌 캐릭터
broll:
  character_description: "futuristic robot character with holographic display, clean tech illustration style"

# 자막: 큰 글씨, 높은 대비
subtitles:
  font_size: 22
```

> 비주얼 세팅의 상세 옵션은 `10-visual-identity.md` 참조.

### 스킬 활용 예시

```
/generate-video news-001
→ 뉴스 대본 기반 슬라이드 + B-roll + TTS + 최종 영상

/generate-shorts news-001
→ 핵심 포인트 1개를 추출하여 60초 쇼츠로 변환
```

### 운영 루틴

| 요일 | 활동 |
|------|------|
| 월/수/금 | 뉴스 리서치 + 대본 작성 + 영상 생성 |
| 화/목 | 쇼츠 생성 + 업로드 |
| 주말 | 다음 주 소재 메모 + 트렌드 모니터링 |

```
주간 생산 흐름:

월 ──→ 뉴스 리서치 + /generate-video
화 ──→ 어제 영상에서 /generate-shorts 추출
수 ──→ 새 뉴스 + /generate-video
목 ──→ 쇼츠 + 지난 영상 성과 분석
금 ──→ 주간 정리 영상 or 심층 분석 영상
```

> **팁:** 뉴스 채널은 속도가 생명입니다. `/generate-video`를 실행해두고 파이프라인이 돌아가는 동안 다음 소재를 리서치하면 시간 효율이 올라갑니다.

> **핵심:** IT 뉴스 채널 = 빠른 TTS(1.15) + 테크 캐릭터 + 속보 톤 대본 + 주 3~5회 루틴. `/detect-releases`로 소재 발굴을 자동화하는 것이 차별점.

## Part 3: 시나리오 B -- 교육/튜토리얼 채널

### 컨셉

초보자를 위한 단계별 가이드, 시리즈 중심 채널입니다. 한 주제를 깊이 있게 다루되, 난이도를 점진적으로 올려가며 시청자를 성장시키는 구조입니다.

### 대본 패턴

```
[인트로 - 질문 + 약속]
코딩을 전혀 몰라도 AI 영상을 만들 수 있다면 어떨까요?
오늘 3단계로 알려드리겠습니다.

[단계 1 - 쉬운 것부터]
첫 번째 단계는 대본 작성입니다.
AI에게 주제만 알려주면 대본을 자동으로 만들어줍니다.

[단계 2 - 중간 난이도]
두 번째 단계는 슬라이드 생성입니다.
대본을 넣으면 디자인된 슬라이드가 자동으로 만들어집니다.

[단계 3 - 핵심]
세 번째 단계는 최종 영상 합성입니다.
음성, 슬라이드, 자막이 하나의 영상으로 합쳐집니다.

[정리 + CTA]
오늘 배운 3단계를 정리하면, 대본 작성, 슬라이드 생성, 영상 합성입니다.
다음 영상에서는 심화 내용을 다루겠습니다.
```

Claude Code 프롬프트:

```
"AI로 첫 유튜브 영상 만들기"를 주제로 교육용 대본을 작성해줘.
초보자 대상, 3단계 구조, 차분한 톤으로.
projects/tutorial-001/script.txt에 저장해줘.
```

### 추천 설정

```yaml
# TTS: 차분하고 이해하기 쉬운 속도
tts:
  elevenlabs:
    speed: 0.95              # 천천히, 이해하기 쉽게
    stability: 0.75          # 적당한 감정 -- 딱딱하지 않게

# 자막
subtitles:
  font_size: 22
```

> 교육 채널의 시각 스타일은 밝은 배경 + 블루 포인트가 기본입니다. `10-visual-identity.md` Part 3의 "교육/튜토리얼 채널" 추천 참조.

### 시리즈 전략

시리즈물은 시청자 유지율이 높습니다. 에피소드 간 연결이 자연스러워야 합니다.

| 시리즈 | 에피소드 구성 |
|--------|-------------|
| AI 영상 입문 | 1. 대본 쓰기 → 2. 첫 영상 만들기 → 3. 쇼츠 만들기 |
| AI 도구 리뷰 | 1. ChatGPT → 2. Claude → 3. Gemini |
| 자동화 마스터 | 1. 기본 파이프라인 → 2. 배치 생산 → 3. 커스텀 스킬 |

배치 제작:

```
"AI 영상 입문" 시리즈 대본 3편을 한번에 작성해줘.
에피소드 간 연결이 자연스럽도록 해줘.
projects/series-intro-001/script.txt
projects/series-intro-002/script.txt
projects/series-intro-003/script.txt
```

> `12-content-strategy.md` 참조 -- 시리즈 기획과 배치 대본 작성의 상세 전략.

### 멀티포맷 활용

교육 채널은 본편 + 캐러셀 조합이 효과적입니다. 쇼츠는 "핵심 한 가지만" 추출하여 본편으로 유도합니다.

```
본편 영상 (유튜브)
  ├── 캐러셀 (인스타) ── 핵심 단계를 카드로 정리
  └── 쇼츠 (유튜브/릴스) ── "핵심 한 가지"만 추출
```

```
/generate-video tutorial-001
→ 본편 영상

/carousel-copywriting tutorial-001
/generate-carousel tutorial-001
→ 핵심 포인트를 카드뉴스로

/generate-shorts tutorial-001
→ "이것만 기억하세요" 60초 쇼츠
```

### 운영 루틴

| 요일 | 활동 |
|------|------|
| 월 | 시리즈 기획 + 대본 작성 |
| 화 | `/generate-video` + 영상 생성 |
| 수 | 캐러셀 + 쇼츠 생성 |
| 목 | 영상 업로드 + 캐러셀 업로드 |
| 금 | 댓글 피드백 → 다음 에피소드 반영 |

> **핵심:** 교육 채널 = 느린 TTS(0.95) + 밝은 배경 + 시리즈 구조 + 멀티포맷. 시리즈 대본을 배치로 작성하면 일관성과 효율이 동시에 올라갑니다.

## Part 4: 시나리오 C -- 트렌드/바이럴 콘텐츠

### 컨셉

화제의 주제를 빠르게 커버하는 쇼츠 중심 채널입니다. 첫 1초 훅이 전부이며, 24시간 내 업로드가 생명입니다.

### 대본 패턴 -- 쇼츠 최적화

쇼츠 대본은 극도로 짧고 임팩트 있어야 합니다. 훅 1문장 + 핵심 3~4문장 + CTA 1문장이 기본 구조입니다.

```
[훅 - 1문장]
GPT-5가 무료로 풀렸습니다.

[핵심 - 3~4문장]
이번에 달라진 점은 딱 하나입니다.
추론 능력이 기존 대비 3배 향상되었습니다.
실제로 테스트해보면 코딩 문제 정답률이 40%에서 87%로 뛰었습니다.

[CTA - 1문장]
여러분도 지금 바로 써보세요. 링크는 댓글에.
```

Claude Code 프롬프트:

```
"GPT-5 무료 공개" 주제로 30초 쇼츠 대본을 작성해줘.
훅 1문장 + 핵심 3문장 + CTA 1문장 구조로.
projects/viral-001/script.txt에 저장해줘.
```

### 추천 설정

```yaml
# TTS: 빠르고 에너지 넘치는 톤
tts:
  elevenlabs:
    speed: 1.15              # 빠른 전달
    stability: 0.70          # 약간의 감정 변화

# 자막: 쇼츠에서는 더 큰 글씨
subtitles:
  font_size: 24              # 모바일에서 잘 보이도록
```

### 바이럴 훅 패턴 5가지

| 패턴 | 예시 |
|------|------|
| 충격 사실 | "GPT-5가 무료로 풀렸습니다" |
| 숫자 임팩트 | "10초 만에 영상 1개가 완성됩니다" |
| 반전 질문 | "코딩 없이 앱을 만들 수 있다면?" |
| 경고/긴급 | "이거 모르면 손해 봅니다" |
| 비교 | "ChatGPT vs Claude, 승자는?" |

```
"바이럴 쇼츠 대본 5개를 만들어줘.
각각 다른 훅 패턴(충격, 숫자, 반전, 경고, 비교)을 사용해줘.
주제: 이번 주 AI 뉴스"
```

### 운영 전략

트렌드 채널의 핵심 공식:

```
소재 발견 (0h) → 대본 (15min) → /generate-shorts (30min) → 업로드 (1h)
─────────────── 총 1시간 이내 ───────────────
```

- 매일 쇼츠 1~2개 + 주 2회 본편
- 쇼츠에서 바이럴 → 본편으로 유도 ("자세한 내용은 본편에서")
- 트렌드 소재는 **24시간 내 업로드**가 생명 -- 늦으면 경쟁자에게 뺏김

### 운영 루틴

| 시간대 | 활동 |
|--------|------|
| 아침 | 트렌드 모니터링 (X, Reddit, HN) |
| 오전 | 소재 선정 + 쇼츠 대본 + `/generate-shorts` |
| 오후 | 업로드 + 성과 확인 + 본편 대본 작성 |
| 저녁 | 본편 `/generate-video` 실행 |

> **핵심:** 트렌드 채널 = 빠른 TTS(1.15) + 큰 자막(24px) + 바이럴 훅 5패턴 + 매일 업로드. 속도가 곧 경쟁력이므로 `/generate-shorts` 파이프라인을 최대한 활용하세요.

## Part 5: 시나리오 D -- AI 활용 사례 채널

### 컨셉

"나도 AI로 이거 할 수 있어" -- 비개발자 대상으로 AI 활용법을 보여주는 채널입니다. 과정을 직접 보여주는 것이 핵심이고, 캐러셀이 주력 포맷입니다.

### 대본 패턴

```
[공감 인트로]
유튜브 시작하고 싶은데, 편집할 시간이 없으시죠?

[솔루션 제시]
AI 자동화 시스템을 쓰면 대본 하나로 영상, 쇼츠, 카드뉴스까지 한번에 만들 수 있습니다.

[과정 보여주기 - 1]
실제로 해보겠습니다. 먼저 대본을 작성합니다.
Claude Code에게 주제만 알려주면 됩니다.

[과정 보여주기 - 2]
대본이 완성되면 한 줄 명령어로 영상을 생성합니다.
슬라이드, 음성, 자막이 자동으로 만들어집니다.

[결과 + CTA]
10분 만에 3가지 콘텐츠가 완성되었습니다.
여러분도 해보세요. 방법은 아래 링크에 있습니다.
```

Claude Code 프롬프트:

```
"AI로 유튜브 콘텐츠 3종 세트 만들기"를 주제로 대본을 작성해줘.
비개발자 대상, 공감 인트로 + 과정 시연 + 결과 보여주기 구조로.
projects/usecase-001/script.txt에 저장해줘.
```

### 캐러셀 중심 전략

이 시나리오는 캐러셀이 주력 포맷입니다. 인스타그램의 "저장" 기능이 도달률을 크게 올려주기 때문입니다.

```
[1] /carousel-copywriting usecase-001
    → 카드별 카피 사전 기획 + 8가지 메트릭 자가평가
    ↓
[2] 카피 검토 및 수정
    ↓
[3] /generate-carousel usecase-001
    → 확정된 카피 기반 TSX 카드 생성
    ↓
[4] 결과 확인 + 업로드
```

> `07-multi-format-strategy.md` 참조 -- 캐러셀 카피라이팅 프리프로덕션 워크플로우의 상세 설명.

### 추천 설정

```yaml
# TTS: 친근하고 차분한 톤
tts:
  elevenlabs:
    speed: 1.0               # 보통 속도
    stability: 0.70          # 자연스러운 감정

# B-roll: 친근한 캐릭터
broll:
  character_description: "friendly character demonstrating on a laptop, clean flat illustration style, warm colors"
```

### 멀티포맷 극대화

하나의 주제로 세 가지 포맷을 모두 만듭니다:

| 포맷 | 역할 | 명령어 |
|------|------|--------|
| 캐러셀 | 핵심 단계를 카드로 정리 (저장률) | `/carousel-copywriting` + `/generate-carousel` |
| 쇼츠 | "이렇게 쉬워요" 과정 요약 (바이럴) | `/generate-shorts` |
| 본편 | 전체 과정 상세 시연 (깊이) | `/generate-video` |

### 운영 루틴

| 요일 | 활동 |
|------|------|
| 월 | 활용 사례 소재 선정 + 대본 |
| 화 | 캐러셀 카피라이팅 + 카드 생성 |
| 수 | 본편 영상 + 쇼츠 생성 |

## Part 6: 나의 시나리오 설계하기

4가지 시나리오를 참고했다면, 이제 나만의 시나리오를 설계할 차례입니다. 아래 워크시트를 채워보세요.

```
[ 나의 채널 시나리오 ]

채널 컨셉: _______________
타겟 시청자: _______________
주력 포맷: [ ] 영상  [ ] 쇼츠  [ ] 캐러셀
업로드 빈도: _______________

대본 패턴:
  인트로 스타일: _______________
  본문 구조: _______________
  아웃트로 CTA: _______________

TTS 속도: ___
슬라이드 분위기: _______________

첫 시리즈 주제 3개:
1. _______________
2. _______________
3. _______________
```

### 설계 팁

| 항목 | 질문 |
|------|------|
| 컨셉 | "내 채널을 한 문장으로 소개하면?" |
| 타겟 | "이 영상을 누가 검색할까?" |
| 포맷 | "시청자가 주로 어떤 기기로 볼까?" (PC=영상, 모바일=쇼츠+캐러셀) |
| 빈도 | "현실적으로 주 몇 회 만들 수 있는가?" |
| TTS | "뉴스처럼 빠르게? 강의처럼 천천히?" |

### Claude Code로 시나리오 검증

워크시트를 채운 후 Claude Code에게 검증을 요청할 수 있습니다:

```
내 채널 시나리오를 검토해줘:
- 컨셉: AI 활용 팁 (비개발자 대상)
- 포맷: 캐러셀 + 쇼츠
- 빈도: 주 3회
- TTS: 1.0
이 설정에서 개선할 점이 있는지 알려줘.
```

**도메인별 시나리오 설계 프롬프트 (복사해서 사용):**

```
나는 "재테크/투자 입문" 채널을 만들려고 해.
타겟: 20~30대 사회 초년생, 주력 포맷: 영상 + 캐러셀.
이 채널에 맞는 TTS 속도, 슬라이드 스타일, 대본 패턴, 주간 운영 루틴을 제안해줘.
추가로 첫 시리즈 3편 주제도 추천해줘.
```

```
나는 "건강/다이어트 팁" 채널을 만들려고 해.
타겟: 30~40대 직장인, 주력 포맷: 쇼츠 + 캐러셀.
B-roll 캐릭터 스타일, 색상 팔레트, 바이럴 훅 패턴을 포함해서 전체 세팅을 제안해줘.
```

```
나는 "소상공인 마케팅" 채널을 만들려고 해.
타겟: 카페/식당/쇼핑몰 사장님, 주력 포맷: 캐러셀 + 쇼츠.
실제 매출 숫자를 강조하는 슬라이드 스타일과 인트로 훅 패턴 3가지를 제안해줘.
```

```
나는 "영어 회화 학습" 채널을 만들려고 해.
타겟: 직장인 영어 학습자, 주력 포맷: 쇼츠(30초 표현) + 영상(5분 강의).
쇼츠는 표현 1개씩, 영상은 관련 표현 5개 묶음. 이 구조에 맞는 전체 세팅을 제안해줘.
```

```
나는 "여행 브이로그 스타일" 채널을 만들려고 해.
타겟: 20~40대 여행 계획 중인 사람, 주력 포맷: 영상 + 캐러셀.
따뜻한 감성의 비주얼 스타일과 코스별 타임라인 슬라이드 레이아웃을 제안해줘.
```

### 시나리오 조합 예시

하나의 시나리오만 고를 필요는 없습니다. 두 시나리오를 조합할 수도 있습니다:

| 조합 | 설명 |
|------|------|
| A+C | 뉴스 본편(A) + 뉴스 요약 쇼츠(C) |
| B+D | 튜토리얼 본편(B) + 활용 사례 캐러셀(D) |
| A+D | 기술 뉴스 영상(A) + 해당 기술 활용법 캐러셀(D) |

> **핵심:** 시나리오는 출발점이지 정답이 아닙니다. 워크시트를 채운 후 2~3주 운영해보고, 시청자 반응에 따라 대본 패턴과 설정을 조정하세요. 스킬로 자산화하면(`10-visual-identity.md` Part 5 참조) 변경 이력이 남아 되돌리기도 쉽습니다.

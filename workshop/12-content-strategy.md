# 소재 선정 & 운영 노하우

> "도구가 아무리 좋아도, 대본이 안 좋으면 아무도 안 봅니다"

---

## Part 1: 대본이 곧 영상 품질

이 시스템에서 여러분이 직접 영향을 미칠 수 있는 가장 중요한 변수는 **대본(script.txt)**입니다.

```
좋은 대본 + 자동 생성 = 좋은 영상
나쁜 대본 + 자동 생성 = 빠르게 만든 나쁜 영상
```

자동화는 **속도**를 줍니다. **품질**은 대본에서 옵니다.

## Part 2: 소재 찾기 3가지 방법

### 방법 1: 트렌드 기반

Claude Code에게 리서치를 맡깁니다:

```
최근 1주일간 AI/자동화 관련 트렌드를 조사해줘.
유튜브 영상 소재로 쓸 만한 주제 5개를 추천해줘.
각 주제별로 왜 지금 만들면 좋은지 이유도 알려줘.
```

**소재 발굴처:**
- X(트위터) 트렌딩 해시태그
- 레딧 인기 게시물
- 구글 트렌드
- 유튜브 인기 급상승 → 관련 키워드

### 방법 2: 리버스 프롬프팅 (05-slide-generation.md 참조)

인기 채널의 영상 구조를 분석합니다:

```
이 유튜브 채널의 최근 인기 영상 3개 대본 구조를 분석해줘.
공통 패턴(인트로 방식, 본문 구성, 아웃트로)을 정리해줘.
```

### 방법 3: 시리즈화

하나의 큰 주제를 여러 각도로 나눕니다:

| 메인 주제 | 시리즈 예시 |
|----------|-----------|
| AI 자동화 | "AI 자동화 입문", "AI로 쇼츠 만들기", "AI 영상 편집" |
| Claude Code | "Claude Code 기초", "스킬 만들기", "실전 활용법" |
| 유튜브 운영 | "알고리즘 이해", "썸네일 전략", "댓글 관리" |
| 재테크 입문 | "ETF란?", "적금 vs 투자", "첫 주식 매수" |
| 건강 루틴 | "아침 스트레칭", "점심 산책", "취침 루틴" |
| 요리 레시피 | "10분 파스타", "도시락 3종", "냉장고 털기" |
| 영어 회화 | "공항 영어", "카페 주문", "비즈니스 이메일" |

> **핵심:** 시리즈물은 **구독 전환율**이 높습니다. 1편을 본 사람이 2편, 3편을 찾게 됩니다.

### 도메인별 소재 리서치 프롬프트 (복사해서 사용)

```
최근 1주일간 AI/자동화 분야에서 유튜브 영상으로 만들면 조회수 잘 나올 소재 5개를 추천해줘.
각 소재별로 제목 후보 2개, 타겟 시청자, 예상 길이를 알려줘.
```

```
"재테크/투자" 분야에서 초보 투자자가 가장 많이 검색하는 키워드 10개를 알려줘.
그중 영상 소재로 쓸 만한 것 5개를 골라서 대본 구조를 간략히 제안해줘.
```

```
"건강/운동" 채널에서 시리즈 3편을 기획하고 싶어.
"직장인 퇴근 후 운동"을 메인 주제로 3편의 부제목과 각 편 핵심 포인트 3개씩 제안해줘.
```

```
"요리/자취" 채널용 콘텐츠 캘린더를 1주일치 만들어줘.
월~금 매일 다른 주제, 각각 3분 분량, 시리즈 연결 가능하게.
```

```
"영어 회화" 채널에서 직장인 대상 콘텐츠를 만들려고 해.
"비즈니스 영어" 시리즈 5편의 에피소드 구성을 제안해줘.
각 편마다 쇼츠 1개, 캐러셀 카드 5장으로 변환 가능한 구조로.
```

## Part 3: 대본 작성 팁

### 구조: 인트로 → 본문 → 아웃트로

```
[인트로] 1~2 문단
  → 시청자의 관심을 끄는 질문 또는 충격적 사실
  → "이거 알고 계셨나요?" "많은 분들이 모르는..."

[본문] 5~8 문단
  → 핵심 포인트 3~5개
  → 각 포인트 = 1~2 문단
  → 구체적 사례, 숫자, 비교

[아웃트로] 1~2 문단
  → 내용 정리
  → CTA (구독, 좋아요, 댓글 유도)
```

### 문단 길이 가이드

| 길이 | 효과 | 추천 상황 |
|------|------|----------|
| 30~50자 | 짧고 임팩트 | 인트로, 전환, 강조 |
| 50~100자 | 균형잡힌 | 본문 (대부분의 문단) |
| 100자 이상 | 설명이 길어짐 | 피하기 (슬라이드가 빈약해 보임) |

### 후킹 패턴

| 패턴 | 예시 |
|------|------|
| 질문형 | "유튜브 영상 하나 만드는데 얼마나 걸리시나요?" |
| 반전형 | "AI 영상 도구는 많습니다. 그런데 이상한 점이 있습니다." |
| 숫자형 | "28일 만에 구독자 2,444명이 늘었습니다." |
| 공감형 | "해보긴 해야 할 것 같은데, 누구 말을 믿어야 할지 모르겠다." |

## Part 4: 배치 제작

### 한 번에 여러 대본 만들기

```
"AI 자동화" 시리즈로 3편 분량의 대본을 만들어줘.
각각 projects/series-001, series-002, series-003에 저장해줘.
```

### 연속 파이프라인 실행

```bash
# 3개 영상 연속 생성
uv run video-automation pipeline script-to-video \
    --input projects/series-001/script.txt --project series-001

uv run video-automation pipeline script-to-video \
    --input projects/series-002/script.txt --project series-002

uv run video-automation pipeline script-to-video \
    --input projects/series-003/script.txt --project series-003
```

> **팁:** `/generate-video`를 먼저 3개 다 실행해 슬라이드를 미리 생성해두고, 파이프라인을 연달아 돌리면 더 효율적입니다.

## Part 5: 1 대본 → 3 콘텐츠 (다중 포맷)

> 멀티포맷 전략의 자세한 가이드는 **07-multi-format-strategy.md**를 참조하세요.

하나의 대본으로 세 가지 포맷을 동시에 만듭니다:

```
script.txt (대본)
    │
    ├── /generate-video    → 16:9 유튜브 본편
    ├── /generate-shorts   → 9:16 쇼츠/릴스
    └── /generate-carousel → 4:5 인스타 카드뉴스
```

### 실행 순서

```bash
# 1. Claude Code에서 각 포맷 스킬 실행
/generate-video my-project
/generate-shorts my-project
/generate-carousel my-project

# 2. 파이프라인 실행
uv run video-automation pipeline script-to-video \
    --input projects/my-project/script.txt --project my-project

uv run video-automation pipeline script-to-shorts \
    --input projects/my-project/script.txt --project my-project

uv run python scripts/regenerate_carousel.py my-project
```

> **핵심:** "오늘 뭘 올리지?"가 아니라 "세 개 중 뭘 먼저 올리지?"로 바뀝니다.

## Part 6: 품질 루프

> 체계적인 품질 개선 방법은 **11-quality-iteration.md**에서 자세히 다룹니다.

```
[1] 첫 영상 제작
    ↓
[2] 결과 확인 → 피드백 정리
    "음성이 너무 빠르다" / "슬라이드가 단조롭다" 등
    ↓
[3] 설정 또는 스킬 개선
    speed: 1.0 → 0.95 / 슬라이드 다양성 규칙 추가
    ↓
[4] 다음 영상 제작 → 품질 상승
    ↓
[반복]
```

이 루프를 반복하면:
- **설정값**이 점점 내 채널에 최적화됩니다
- **Skill**이 점점 정교해집니다
- 같은 노력으로 **더 나은 결과물**이 나옵니다

> 이것이 "도구를 쓰는 사람"과 "도구를 만드는 사람"의 차이입니다.
> 나만의 채널 디자인 시스템 구축은 **10-visual-identity.md**, 채널 유형별 실전 가이드는 **14-real-world-scenarios.md**를 참조하세요.

## Part 7: ElevenLabs 목소리 팁

### 한국어 음성 선택 기준

| 기준 | 설명 |
|------|------|
| 발음 정확도 | 한글 받침 처리가 자연스러운지 |
| 감정 범위 | 단조롭지 않고 적절한 억양 변화 |
| 속도 호환성 | speed 설정 변경 시 자연스러운지 |

### 크레딧 관리

- ElevenLabs 무료 티어: 월 10,000자
- 영상 1개 (10문단, 약 800자): ~800자 소모
- **무료 티어로 약 12편** 생성 가능

### 2차 API 키 (크레딧 소진 대비)

```yaml
# config/config.api.yaml
tts:
  elevenlabs:
    api_key_env: "ELEVENLABS_API_KEY"
    fallback_api_key_env: "ELEVENLABS_SECOND_API_KEY"
```

`.env`에 두 번째 계정 키를 추가하면, 첫 번째 키의 크레딧이 소진됐을 때 자동으로 전환됩니다.

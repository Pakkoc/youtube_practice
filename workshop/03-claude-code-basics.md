# Claude Code 기초 가이드

> 클로드 코드를 처음 쓰는 분을 위한 안내서

---

## Part 1: Claude Code란?

**한 줄 정의:** 터미널(검은 화면)에서 동작하는 AI 코딩 어시스턴트

일반 ChatGPT와 다른 점:
- **대화만 하는 게 아니라, 실제로 파일을 읽고 쓰고 명령어를 실행**합니다
- 여러분의 컴퓨터에서 직접 작업합니다
- 코드를 작성하고, 수정하고, 삭제할 수 있습니다

> **비유:** ChatGPT는 "전화 상담원" — 물어보면 답해줍니다. Claude Code는 "원격 조종 개발자" — 물어보면 직접 해줍니다.

## Part 2: 기본 실행법

### 2-1. 시작하기

프로젝트 폴더에서 터미널을 열고:

```bash
claude --dangerously-skip-permissions
```

> `--dangerously-skip-permissions`란? Claude Code가 파일을 읽거나 쓸 때 매번 "허락하시겠습니까?"를 묻지 않는 옵션입니다. 워크숍에서는 편의상 사용합니다.

> **TIP: `cc` 별칭(alias) 설정으로 1초 만에 실행하기**
>
> 매번 긴 명령어를 치는 건 번거롭습니다. 터미널 설정 파일에 별칭을 등록해 두면 `cc`만 입력하면 됩니다:
>
> ```bash
> # Mac/Linux — .zshrc에 추가 (bash 사용자는 .bashrc)
> echo "alias cc='claude --dangerously-skip-permissions'" >> ~/.zshrc
> source ~/.zshrc
>
> # Windows (Git Bash) — .bashrc에 추가
> echo "alias cc='claude --dangerously-skip-permissions'" >> ~/.bashrc
> source ~/.bashrc
> ```
>
> 이제 프로젝트 폴더에서 `cc`만 치면 바로 시작됩니다.
>
> `source` 명령은 설정 파일의 변경 사항을 현재 터미널에 즉시 적용합니다. 새 터미널을 열면 자동으로 적용되므로 한 번만 하면 됩니다.
>
> **주의:** 이름에 `dangerously`가 들어 있는 만큼, 무엇을 허용하는지 이해한 상태에서 사용하세요. 워크숍 환경에서는 문제없지만, 실무에서는 상황에 맞게 판단하시기 바랍니다.

실행하면 대화창이 나타납니다:

```
╭─────────────────────────────────────╮
│ Claude Code                          │
│                                      │
│ Type a message...                    │
╰─────────────────────────────────────╯
```

### 2-2. 종료하기

대화창에서 아래 중 하나를 입력합니다:
- `/exit` 또는 `Ctrl + C`

## Part 3: Claude Code가 할 수 있는 것

### 파일 읽기

```
projects/workshop-001/script.txt를 읽어줘
```
```
output 폴더에 어떤 파일이 있는지 보여줘
```

### 파일 작성

```
projects/workshop-001/script.txt에 이 내용으로 대본을 작성해줘:

AI가 유튜브 영상을 자동으로 만드는 시대가 왔습니다.

첫 번째 방법은 대본을 작성하는 것입니다.
```

### 파일 수정

```
script.txt의 3번째 문단을 수정해줘. 더 짧고 임팩트 있게.
```

```

```

### 코드/시스템 분석

```
현재 TTS 설정이 어떻게 되어 있는지 알려줘
```

### 슬래시 명령어

```
/generate-video workshop-001
/generate-shorts workshop-001
/generate-carousel workshop-001
```

## Part 4: 대화형 작업의 핵심

### 맥락 유지

Claude Code는 세션 안에서 이전 대화를 기억합니다:

```
나: "script.txt를 읽어줘"
Claude: (파일 내용 표시)
나: "3번째 문단을 줄여줘"          ← "아까 그 파일"을 기억함
Claude: (수정 완료)
나: "다시 읽어줘"                  ← 수정된 결과 확인
```

### 연쇄 작업

여러 단계를 한 번에 지시할 수 있습니다:

```
script.txt를 읽고, 10개 문단으로 나눠져 있는지 확인하고,
문단 수가 부족하면 추가해줘.
```

### 수정 반복

결과가 마음에 안 들면 계속 수정할 수 있습니다:

```
나: "5번 슬라이드 배경색 바꿔줘"
Claude: (수정 완료)
나: "아니, 더 어둡게"
Claude: (재수정 완료)
나: "좋아, 이번엔 텍스트도 노란색으로"
```

## Part 5: 워크숍에서 자주 쓸 프롬프트

| 상황 | 프롬프트 예시 |
|------|-------------|
| 대본 작성 | `"AI 자동화"를 주제로 3분짜리 유튜브 대본을 써줘` |
| 대본 수정 | `2번째 문단이 너무 길어. 반으로 줄여줘` |
| 영상 생성 | `/generate-video workshop-001` |
| 쇼츠 생성 | `/generate-shorts workshop-001` |
| 카드뉴스 | `/generate-carousel workshop-001` |
| 설정 확인 | `현재 TTS 설정 보여줘` |
| 오류 해결 | `이 에러를 해결해줘: [에러 메시지 붙여넣기]` |
| 파일 확인 | `프로젝트 폴더 구조를 보여줘` |
| 발음 사전 | `발음 사전에 "GitHub=깃허브" 추가해줘` |
| 슬라이드 수정 | `3번 슬라이드를 리스트 레이아웃으로 바꿔줘` |

### 도메인별 대본 프롬프트 (복사해서 주제만 바꾸세요)

**IT/테크:**
```
"2025년 개발자가 꼭 알아야 할 AI 코딩 도구 TOP 5"를 주제로
3분짜리 유튜브 대본 써줘. 각 도구마다 한 문단, 비교 포인트 포함.
projects/workshop-001/script.txt에 저장.
```

**재테크/경제:**
```
"월급 200만원으로 시작하는 ETF 투자 로드맵"을 주제로
3분짜리 유튜브 대본 써줘. 초보 투자자 대상, 구체적 금액과 수익률 예시 포함.
projects/workshop-001/script.txt에 저장.
```

**건강/운동:**
```
"하루 10분, 사무실에서 할 수 있는 스트레칭 루틴"을 주제로
3분짜리 유튜브 대본 써줘. 직장인 대상, 동작마다 문단 분리.
projects/workshop-001/script.txt에 저장.
```

**요리/라이프스타일:**
```
"자취생을 위한 10분 원팟 파스타 3가지"를 주제로
3분짜리 유튜브 대본 써줘. 재료 목록 + 과정 설명 구조.
projects/workshop-001/script.txt에 저장.
```

**교육/어학:**
```
"영어 발음이 한국인스러운 3가지 습관과 교정법"을 주제로
3분짜리 유튜브 대본 써줘. 성인 학습자 대상, Before/After 예문 포함.
projects/workshop-001/script.txt에 저장.
```

**마케팅/창업:**
```
"인스타그램 릴스로 월 100만원 매출 만든 과정"을 주제로
3분짜리 유튜브 대본 써줘. 소상공인 대상, 주차별 실행 단계와 구체적 숫자 포함.
projects/workshop-001/script.txt에 저장.
```

**여행:**
```
"일본 오사카 2박 3일, 50만원으로 완벽한 코스"를 주제로
3분짜리 유튜브 대본 써줘. 첫 일본 여행자 대상, 날짜별 코스와 예산 배분 포함.
projects/workshop-001/script.txt에 저장.
```

> **집에서 활용 팁:** 위 프롬프트를 그대로 복사해서 주제/대상/길이만 바꾸면 어떤 분야든 대본을 만들 수 있습니다.

### 슬라이드 수정 프롬프트 (집에서 고도화용)

```

```

```

## Part 7: 알아두면 좋은 것

### 세션과 기억

- Claude Code는 **한 세션 안에서** 이전 대화를 기억합니다
- 터미널을 닫으면 세션이 끝나고 기억이 초기화됩니다
- 하지만 **파일에 저장된 것**은 남아 있으므로 걱정 없습니다

### 권한 시스템

워크숍에서는 `--dangerously-skip-permissions`으로 권한을 건너뛰지만, 실제 운영할 때는:
- 파일 수정/삭제 시 확인 프롬프트가 표시됩니다
- `Y`를 눌러 허락하거나, `N`으로 거부할 수 있습니다
- 중요한 파일을 실수로 삭제하는 것을 방지합니다

### 비용

- Claude Code 사용 자체는 **Anthropic API 크레딧**을 소모합니다
- 워크숍 중에는 충분한 크레딧이 있으면 됩니다
- 사용량은 [console.anthropic.com](https://console.anthropic.com/)에서 확인 가능

> **핵심:** Claude Code는 "대화하면서 작업하는 도구"입니다. 코딩을 몰라도 자연어로 지시하면 됩니다. 결과가 마음에 안 들면 다시 말하면 됩니다.

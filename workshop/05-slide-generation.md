# 슬라이드 생성 & 리버스 프롬프팅 가이드

> "코드를 몰라도 됩니다. Claude Code가 코드를 쓰고, Remotion이 영상으로 바꿉니다."

---

## Part 1: Remotion TSX란?

### 한 줄 설명
**디자인을 코드로 표현한 파일**입니다. Remotion이라는 도구가 이 코드를 읽고 영상/이미지로 변환합니다.

> **비유:** TSX = 건축 설계도 / Remotion = 설계도대로 건물을 짓는 건설회사
> 여러분은 설계도를 직접 그릴 필요 없습니다. Claude Code가 그려줍니다.

### 왜 코드로 슬라이드를 만드나?

| 이유 | 설명 |
|------|------|
| **AI 친화적** | AI는 텍스트(코드)를 아주 잘 다룹니다. 이미지보다 코드가 정확도 높음 |
| **수정이 쉬움** | "색상 바꿔줘" 한마디면 전체 슬라이드 일괄 수정 |
| **일관된 스타일** | 디자인 토큰(색상, 폰트, 간격)이 코드로 관리되어 통일감 유지 |
| **애니메이션 내장** | 페이드인, 스프링 효과, 카메라 이동 등이 코드에 포함 |
| **재현 가능** | 같은 코드 = 같은 결과. 랜덤 요소 없음 |

## Part 2: 슬라이드 생성 워크플로

### 전체 과정 (5단계)

```
[1] 대본 분석
    → Claude Code가 각 문단의 콘텐츠 유형 파악
    → "이 문단은 비교 내용", "이 문단은 숫자 통계" 등

[2] 아트 디렉션
    → 전체 색상 팔레트, 분위기, 스타일 결정
    → "어두운 테마 + 파란색 포인트 + 미니멀 스타일"

[3] 레이아웃 선택
    → 각 슬라이드에 적합한 패턴 선택
    → "문단 1은 큰 제목", "문단 3은 비교 차트" 등

[4] TSX 코드 작성
    → Claude Code가 슬라이드별 디자인 코드 생성
    → slides/001.tsx, slides/002.tsx, ...

[5] 검증
    → TypeScript 문법 체크
    → 렌더링 테스트 (프레임 0 스틸컷)
```

### 여러분이 하는 일

```
/generate-video my-project
```

이 **한 줄**이면 1~5 과정이 자동으로 진행됩니다.

## Part 3: 슬라이드 레이아웃 종류

Claude Code는 대본 내용에 따라 적합한 레이아웃을 자동 선택합니다.

### 16:9 영상 (1920x1080) 레이아웃

| 카테고리 | 패턴 예시 | 언제 사용 |
|---------|----------|----------|
| **텍스트 중심** | 큰 제목, 번호 리스트, 인용문 | 핵심 메시지, 포인트 나열 |
| **다이어그램** | 바 차트, 타임라인, 스텝 표시기 | 데이터, 프로세스 설명 |
| **UI 모형** | 앱 화면, 채팅 UI, 서비스 화면 | 서비스/도구 소개 |
| **메타포** | 피플 픽토그램, 저울, 화살표 흐름 | 비교, 변화, 관계 표현 |

### 9:16 쇼츠 (1080x1920) 레이아웃

| 영역 | 크기 | 용도 |
|------|------|------|
| 상단 | 훅 타이틀 영역 | 시선 끌기 (큰 글씨) |
| 중앙 | 1080x1280 콘텐츠 영역 | 핵심 내용 |
| 하단 | 자막 영역 | 단어별 실시간 하이라이트 |

### 4:5 카드뉴스 (1080x1350) 레이아웃

- 표지 카드 (제목 + 부제)
- 본문 카드 (리스트, 인용, 비교)
- 마무리 카드 (CTA + 팔로우 유도)

## Part 4: 리버스 프롬프팅

> 11:00~12:00 세션의 핵심 내용입니다.

### 리버스 프롬프팅이란?

기존에 잘 만든 영상을 보고 **"이 영상은 어떤 구조로 만들어졌을까?"** 역추적하는 방법입니다.

```
일반 프롬프팅:  "영상 만들어줘" → 결과물 (불확실)
리버스 프롬프팅: 잘된 영상 분석 → 구조 파악 → 그 구조로 새 영상 → 결과물 (예측 가능)
```

### 방법 1: 대본 구조 분석

인기 유튜브 영상의 대본을 가져와서:

```
이 대본의 구조를 분석해줘.
각 문단의 역할(인트로/포인트/전환/아웃트로)을 분류하고,
어떤 슬라이드 레이아웃이 적합한지 추천해줘.
```

### 방법 2: 레퍼런스 영상 기반

```
이 영상의 슬라이드 스타일을 분석해줘:
- 배경색은 어떤 계열인지
- 텍스트 배치는 어떤 패턴인지
- 전환 효과는 어떤 것을 사용하는지

이 스타일로 내 대본의 슬라이드를 만들어줘.
```

### 방법 3: 채널 스타일 패턴화

```
내가 원하는 채널 스타일:
- 배경: 짙은 남색 (#0a0a1a)
- 포인트 색상: 시안 (#00d4ff)
- 텍스트: 흰색, 굵은 글씨
- 분위기: 테크/미래 느낌

이 스타일로 "AI 자동화" 주제의 슬라이드를 만들어줘.
```

### 활용 흐름

```
1. 마음에 드는 유튜브 영상 3~5개 수집
2. Claude Code에게 각 영상의 대본 구조/슬라이드 스타일 분석 요청
3. 공통 패턴 추출 (색상, 레이아웃, 전환 방식)
4. 그 패턴을 Skill로 저장 (/skill-creator)
5. 이후에는 한 줄로 내 채널 스타일 영상 생성
```

> **핵심:** 리버스 프롬프팅의 최종 목표는 **나만의 Skill을 만드는 것**입니다. 분석한 패턴을 Skill로 저장하면, 다음부터는 분석 없이 바로 적용됩니다.
> 채널 디자인 시스템 구축은 **10-visual-identity.md**, 스킬 제작 실습은 **13-custom-skills.md**를 참조하세요.

### 디자인 디테일 수정 프롬프트 (집에서 고도화용)

아래 프롬프트들은 집에서 슬라이드를 세밀하게 다듬을 때 복사해서 쓸 수 있습니다.

**배경/색상:**
```
3번 슬라이드 배경을 단색 대신 좌상단→우하단 그라데이션(#0a0a1a → #1a1a3e)으로 바꿔줘
```

```
5번 슬라이드 배경에 미세한 도트 패턴(opacity 0.05)을 추가해서 밋밋한 느낌을 없애줘
```

```
텍스트 강조 부분에 #00d4ff 단색 대신 그라데이션(#00d4ff → #7c3aed)을 적용해줘
```

**텍스트/타이포그래피:**
```
2번 슬라이드 제목을 80px → 96px로 키우고 자간을 -2px로 조여서 임팩트 높여줘
```

```
6번 슬라이드 본문 줄간격을 1.6 → 1.8로 늘려서 가독성 개선해줘
```

```
7번 슬라이드에서 핵심 숫자 "87%"를 120px Bold, 나머지 텍스트는 32px로 대비시켜줘
```

```
4번 슬라이드의 핵심 키워드 3개에 배경 하이라이트(#00d4ff, opacity 0.2)를 형광펜처럼 적용해줘
```

**레이아웃 고급:**
```
3번 슬라이드를 3단 칼럼으로 바꿔줘. 왼쪽: 아이콘, 가운데: 제목+설명, 오른쪽: 수치.

```

```
5번 슬라이드를 카드 UI 스타일로 바꿔줘. 3개 카드 수평 배치, 각 카드에 아이콘+제목+한줄 설명.
```

```
8번 슬라이드를 가로 바 차트로 바꿔줘. 3개 항목 비교, 포인트 색상 그라데이션 바, 바 끝에 퍼센트 표시.
```

```
9번 슬라이드를 스텝 프로세스(1→2→3→4) 가로 흐름도로 바꿔줘.
각 스텝은 원형 번호 + 아래 텍스트, 스텝 간 화살표 연결.
```

### 수정 후 반영

```bash
# 슬라이드만 재렌더링
uv run python scripts/regenerate_slides.py my-project

# 특정 슬라이드만 재렌더링 (수정한 것만 빠르게 확인)
uv run python scripts/regenerate_slides.py my-project 003 005

# 영상 재합성
uv run python scripts/recompose_video.py my-project
```

> **팁:** 슬라이드 수정 → `regenerate_slides.py` → `recompose_video.py` 순서로 실행하면 전체 파이프라인을 다시 돌리지 않아도 됩니다. 특정 번호 지정 시 30초면 확인 가능.

## Part 6: 디자인 시스템 (참고)

이 시스템의 슬라이드는 통일된 디자인 시스템을 따릅니다:

| 요소 | 기본값 |
|------|--------|
| 해상도 | 1920x1080 (16:9) |
| 폰트 | Pretendard (한국어 + 영문) |
| 배경 | 어두운 톤 + 그라데이션 |
| 포인트 색상 | 시안/민트 계열 |
| 애니메이션 | Spring 기반 (부드러운 물리 효과) |
| 여백 | 상80 / 우120 / 하80 / 좌120 (px) |

> Claude Code가 이 디자인 시스템을 자동으로 따릅니다. 별도로 지정하지 않아도 일관된 스타일이 유지됩니다.

# 시스템 아키텍처 가이드

> 비개발자도 이해할 수 있는 영상 자동화 시스템 구조

---

## Part 1: 비유로 이해하기 — 영상 공장

이 시스템은 **영상 공장**입니다.

```
[원료 투입]          [각 부서가 작업]              [완성품 출하]
script.txt  ──→  음성부서 + 디자인부서 + 이미지부서  ──→  final_video.mp4
 (대본)              (동시에 작업)                    (영상)
```

- **원료** = 대본 (script.txt)
- **각 부서** = features (음성, 슬라이드, 배경이미지 등)
- **공장 관리자** = pipeline (작업 순서 관리)
- **완성품** = final_video.mp4

여러분은 **원료(대본)만 넣으면** 됩니다. 나머지는 공장이 알아서 합니다.

## Part 2: 핵심 폴더 5개만 알면 됩니다

```
프로젝트 루트/
│
├── projects/          ← ★ 여러분의 작업 공간
│   └── workshop-001/
│       ├── script.txt       ← 대본 (여러분이 작성)
│       ├── audio/            ← AI 음성 파일들
│       ├── slides/           ← 슬라이드 코드 + 영상
│       ├── broll/            ← 배경 이미지들
│       └── output/
│           └── final_video.mp4  ← 최종 영상
│
├── config/            ← ★ 설정 파일 (목소리, 스타일 등)
│   ├── config.api.yaml      ← API 모드 설정 (워크숍용)
│   ├── config.base.yaml     ← GPU 모드 설정
│   └── tts_dictionary.yaml  ← TTS 발음 사전
│
├── .claude/skills/    ← ★ Claude Code 스킬 파일
│   ├── slide-generation/    ← 슬라이드 생성 스킬
│   ├── broll/               ← B-roll 생성 스킬
│   ├── tts/                 ← TTS 스킬
│   └── ... (16개 스킬)
│
├── remotion/          ← ★ 영상 렌더링 엔진 (React 기반)
│
├── scripts/           ← ★ 유틸리티 (재렌더링, 재합성 등)
│
└── (나머지 시스템 폴더들 — 직접 건드릴 일 없음)
    ├── app/           ← CLI 명령어 정의
    ├── pipelines/     ← 파이프라인 로직
    ├── features/      ← 개별 기능 모듈
    ├── entities/      ← 데이터 모델
    └── shared/        ← 공유 유틸리티
```

> **핵심:** 위의 ★ 표시된 5개 폴더만 알면 충분합니다. 나머지는 시스템 내부이므로 신경 쓰지 않아도 됩니다.

## Part 3: 프로젝트 디렉토리 — 여러분의 작업 공간

하나의 영상 = 하나의 프로젝트 폴더입니다.

```
projects/workshop-001/
├── script.txt                  ← [입력] 여러분이 작성하는 대본
├── paragraphs/                 ← 문단별 분리 (시스템 자동)
│   ├── 001.txt
│   ├── 002.txt
│   └── ...
├── audio/                      ← TTS 음성 (ElevenLabs가 생성)
│   ├── 001.wav
│   └── ...
├── slides/                     ← 슬라이드 (Claude Code가 작성 → Remotion이 렌더링)
│   ├── 001.tsx                 ← 디자인 코드
│   ├── 001.mp4                 ← 영상 클립
│   └── ...
├── broll/                      ← 배경 이미지 (Gemini가 생성)
│   ├── prompts.json
│   └── generated/
│       ├── broll_001.png
│       └── ...
└── output/                     ← [출력] 최종 결과물
    ├── video_raw.mp4           ← 자막 없는 영상
    ├── corrected_subtitles.srt ← 자막 파일
    ├── video_with_subtitles.mp4
    └── final_video.mp4         ← 최종 영상
```

| 폴더/파일 | 누가 만드나 | 설명 |
|-----------|------------|------|
| `script.txt` | **여러분** | 영상 대본 (또는 AI가 생성) |
| `paragraphs/` | 시스템 자동 | 빈 줄 기준으로 문단 분리 |
| `audio/` | ElevenLabs API | 문단별 음성 파일 |
| `slides/` | Claude Code + Remotion | TSX 코드 → MP4 영상 클립 |
| `broll/` | Gemini API | AI 생성 배경 이미지 |
| `output/` | FFmpeg | 최종 합성 영상 |

## Part 4: 파이프라인 흐름도 (script-to-video)

```
                                    ┌─────────────────────┐
                                    │  음성 합성            │
                                    │  (ElevenLabs)       │
                                    └──────────┬──────────┘
                                               │
┌──────────┐    ┌──────────┐       ┌───────────┤
│  대본     │──→│ 문단 분리  │──→   │           │
│ script.txt│    └──────────┘       │  동시에   │
└──────────┘                        │  3가지    ├─────────────────────┐
                                    │  생성     │                     │
                                    │           │    ┌──────────────┐ │
                                    └───────────┤    │ 슬라이드 수집  │ │
                                                │    │ (Claude Code) │ │
                                                │    └──────┬───────┘ │
                                                │           │         │
                                                │    ┌──────┴───────┐ │
                                                │    │ 배경 이미지    │ │
                                                │    │ (Gemini)     │ │
                                                │    └──────────────┘ │
                                                │                     │
                                                ▼                     │
                                    ┌─────────────────────┐          │
                                    │ 슬라이드 렌더링       │◄─────────┘
                                    │ (Remotion)          │
                                    └──────────┬──────────┘
                                               │
                                    ┌──────────▼──────────┐
                                    │ 영상 합성            │
                                    │ (FFmpeg)            │
                                    └──────────┬──────────┘
                                               │
                                    ┌──────────▼──────────┐
                                    │ 자막 생성 + 합성      │
                                    └──────────┬──────────┘
                                               │
                                    ┌──────────▼──────────┐
                                    │  final_video.mp4    │
                                    └─────────────────────┘
```

> **핵심:** 음성, 슬라이드, 배경 이미지가 **동시에** 만들어지기 때문에 시간이 절약됩니다.

## Part 5: 5가지 파이프라인

이 시스템에는 5가지 "생산 라인"이 있습니다:

| 파이프라인 | 입력 | 출력 | 용도 |
|-----------|------|------|------|
| **script-to-video** | 대본 | 16:9 영상 | 유튜브 본편 |
| **script-to-shorts** | 대본 | 9:16 쇼츠 | 유튜브 쇼츠, 인스타 릴스 |
| **video-to-shorts** | 기존 영상 | 9:16 쇼츠 | 기존 영상 재가공 |
| **script-to-carousel** | 대본 | 카드뉴스 PNG | 인스타그램 캐러셀 |
| **add-subtitles** | 영상 | 자막 있는 영상 | 외부 영상 자막 추가 |

> 워크숍에서는 **script-to-video** (오전)과 **script-to-shorts**, **script-to-carousel** (오후)을 사용합니다.

## Part 6: 핵심 외부 서비스 역할표

| 서비스 | 역할 | 비유 | API 모드 사용 |
|--------|------|------|:------------:|
| **Claude Code** | 슬라이드 디자인 코드 작성, 시스템 제어 | AI 디자이너 + 공장 관리자 | O |
| **ElevenLabs** | 대본 → 자연스러운 음성 합성 | AI 성우 | O |
| **Gemini (Google)** | 슬라이드 배경 이미지 생성 | AI 일러스트레이터 | O |
| **OpenAI Whisper** | 음성 → 텍스트 변환 (쇼츠에만) | AI 속기사 | O (쇼츠만) |
| **Remotion** | TSX 코드 → 영상/이미지 렌더링 | 렌더링 기계 | O (로컬) |
| **FFmpeg** | 영상 합성, 자막 합성, 편집 | 영상 편집 도구 | O (로컬) |

> Remotion과 FFmpeg은 여러분의 컴퓨터에서 로컬로 실행됩니다. API 비용이 들지 않습니다.

## Part 7: "도구를 쓰는 사람" vs "도구를 만드는 사람"

이 시스템의 핵심 차이점은 **Skills 시스템**입니다.

### 도구를 쓰는 사람
- 남이 만든 도구를 그대로 사용
- 도구가 업데이트되면 처음부터 다시 배워야 함
- 자기 채널에 맞는 커스터마이징이 어려움

### 도구를 만드는 사람 (이 시스템)
- **나만의 Skill을 만들어서** Claude Code에게 행동 규칙을 지시
- Skill을 개선할수록 결과물 품질이 올라감
- 채널 스타일, 대본 패턴, 디자인 규칙을 Skill로 자산화

> **핵심:** 이 시스템을 쓴다는 것은 "도구를 쓰는 것"이 아니라 **"나만의 도구를 만드는 것"**입니다. 시간이 갈수록 차이가 벌어집니다.



> **더 깊이 알고 싶다면:** 이 시스템의 아키텍처 원리(FSD, SDD, DDD)와 Claude Code 개발 방법론은 **[18-dev-methodology.md](18-dev-methodology.md)**에서 다룹니다.

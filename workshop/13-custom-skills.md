# 나만의 스킬 만들기 워크숍

> /skill-creator (Claude Code 내장 스킬)로 자동화 레시피 제작 + 실전 실습 3가지

> 난이도: 심화 | 소요: 읽기 15분 + 실습 40분

---

## Part 1: 왜 스킬을 만드나?

반복 작업을 매번 길게 설명하는 대신, 한 줄 명령어로 실행할 수 있습니다. 스킬은 여러분의 규칙과 취향을 코드로 저장한 것입니다.

채널 스타일 일관성도 스킬이 지켜줍니다. "배경은 항상 다크, 포인트는 시안"을 매번 말하는 대신, 스킬에 한번 정의해두면 이후 모든 영상 생성에 자동 적용됩니다.

```
스킬 없이                          스킬 있으면
───────────                      ───────────
"배경은 다크,                     /generate-video my-project
 포인트는 시안,                   → 스킬이 자동으로 규칙 적용
 폰트는 Pretendard Bold 72px,
 애니메이션은 spring(1, 15),
 파스텔은 쓰지 마..."
(매번 반복)
```

시간이 지날수록 스킬은 쌓이는 자산이 됩니다. 경험이 코드가 되고, 시스템이 "나에게 맞게" 진화합니다. "도구를 쓰는 사람"에서 "도구를 만드는 사람"으로 바뀌는 것이 이 단계입니다 (`02-architecture.md` Part 7 참조).

> **핵심:** 스킬 = 반복 작업의 자동화 + 채널 스타일의 일관성 + 경험의 자산화. 만들수록 시스템이 나에게 맞춰집니다.

## Part 2: 좋은 스킬의 3가지 조건

모든 스킬이 동일한 효과를 내지는 않습니다. 잘 만든 스킬은 세 가지 조건을 충족합니다.

| 조건 | 설명 | 좋은 예시 | 나쁜 예시 |
|------|------|----------|----------|
| 구체적 활성화 조건 | 언제 이 스킬이 로드되어야 하는지 명확 | "슬라이드 생성 요청 시" | "영상 관련 작업 시" |
| 명확한 행동 규칙 | 해야 할 것과 하지 말아야 할 것 | "항상 다크 테마, 파스텔 금지" | "예쁘게 해줘" |
| 검증 가능한 결과 | 성공/실패를 판단할 수 있는 기준 | "TSX 컴파일 통과, 최소 3가지 레이아웃" | "좋은 슬라이드" |

```
좋은 스킬의 구조:

활성화 조건  ──→  행동 규칙  ──→  결과 검증
"언제?"          "무엇을?"       "잘 됐나?"
   │                │               │
구체적 키워드    DO/DON'T 목록    측정 가능한 기준
```

> **팁:** 스킬의 활성화 조건이 너무 넓으면 불필요한 상황에서도 로드됩니다. "영상 관련 작업 시"보다 "슬라이드 생성 요청 시"가 정확합니다.

> **핵심:** 구체적 활성화 조건 + 명확한 행동 규칙 + 검증 가능한 결과. 이 세 가지가 갖춰져야 실제로 쓸 수 있는 스킬이 됩니다.

## Part 3: /skill-creator 사용법

스킬은 직접 파일을 만들지 않습니다. `/skill-creator` (Claude Code 내장 스킬 -- 프로젝트 커맨드가 아닌 Claude Code 자체 기능)가 구조화된 스킬 파일을 자동으로 생성합니다.

### 단계별 과정

```
1. Claude Code 대화창에 입력:
   /skill-creator

2. Claude Code가 물어볼 것들:
   - 스킬 이름
   - 무엇을 자동화하고 싶은지
   - 언제 활성화되어야 하는지
   - 구체적인 규칙들

3. Claude Code가 생성하는 것:
   .claude/skills/<skill-name>/
   ├── SKILL.md              ← 메인 인덱스
   └── chapter-*.md          ← 세부 규칙 (필요시)
```

### 핵심 규칙

| 규칙 | 이유 |
|------|------|
| 스킬 파일을 직접 편집하지 않는다 | `/skill-creator`가 구조를 관리함 |
| 400줄 이하로 유지 | 넘으면 chapter 파일로 자동 분리 |
| 활성화 키워드를 명확히 지정 | 불필요한 로드 방지 |

> **팁:** `/skill-creator` (Claude Code 내장 스킬)를 실행하면 대화형으로 진행됩니다. 규칙을 한번에 다 말해도 되고, 질문에 하나씩 답해도 됩니다.

## Part 4: 실습 1 -- 채널 스타일 스킬

**목표:** 내 채널만의 색상/폰트/레이아웃 규칙을 스킬로 만들기

### Step 1: /skill-creator 실행

```
/skill-creator
```

### Step 2: 스킬 정의

```
"내 채널 'TechBrief'의 슬라이드 스타일 스킬을 만들어줘.

규칙:
1. 배경: 항상 다크 (#0a0a1a ~ #0f1923 범위)
2. 포인트 색상: 시안 (#00d4ff)
3. 레이아웃: 텍스트 중심, 다이어그램 선호
4. 제목: Pretendard Bold 72px
5. 본문: Pretendard Regular 36px
6. 애니메이션: spring(mass: 1, damping: 15) 통일
7. 금지: 파스텔톤, 밝은 배경, 둥근 모서리(radius 12 이하)

활성화 조건: /generate-video 또는 슬라이드 생성 요청 시"
```

### Step 3: 생성 결과 확인

```
.claude/skills/techbrief-style/
├── SKILL.md                  ← 색상, 폰트, 금지 사항 정의
└── chapter-layout-rules.md   ← 레이아웃 패턴 상세
```

### Step 4: 테스트

```
/generate-video test-style
→ 생성된 슬라이드가 규칙을 따르는지 확인
```

테스트 체크리스트:

| 확인 항목 | 기대값 |
|----------|--------|
| 배경색 | `#0a0a1a` ~ `#0f1923` 범위 |
| 포인트색 | `#00d4ff` |
| 제목 크기 | 72px |
| 파스텔톤 사용 | 없음 |
| 둥근 모서리 | radius 12 초과만 허용 |

> `10-visual-identity.md` Part 5에서 스타일 스킬 자산화의 전체 맥락을 참조할 수 있습니다.

> **핵심:** 채널 스타일 스킬 = 한번 정의하면 모든 영상에 자동 적용. 매번 색상/폰트를 지정할 필요가 없어집니다.

## Part 5: 실습 2 -- 대본 템플릿 스킬

**목표:** 특정 포맷의 대본을 자동 생성하는 스킬

### Step 1: 스킬 정의

```
/skill-creator

"'3분 뉴스 요약' 대본 템플릿 스킬을 만들어줘.

규칙:
1. 총 8~10 문단 (3분 분량)
2. 인트로: 속보 톤 질문형 ("오늘 ~가 발표되었습니다. 핵심 N가지를 정리합니다.")
3. 본문: 핵심 포인트 3개, 각 2문단
4. 각 포인트: 사실 진술 → 의미/영향 분석
5. 아웃트로: 1문단, 구독 CTA
6. 문단당 50~80자 (짧고 임팩트)

활성화 조건: '뉴스 대본', '속보 대본', '3분 요약' 키워드 시"
```

### Step 2: 생성 결과 확인

```
.claude/skills/news-script-template/
└── SKILL.md          ← 대본 구조, 문단 길이, 톤 규칙
```

### Step 3: 테스트

```
"GPT-5 발표" 주제로 뉴스 대본 써줘
→ 스킬이 자동 적용되어 뉴스 형식 대본 생성
```

테스트 체크리스트:

| 확인 항목 | 기대값 |
|----------|--------|
| 총 문단 수 | 8~10개 |
| 인트로 톤 | 속보 형 ("~가 발표되었습니다") |
| 핵심 포인트 | 3개, 각 2문단 |
| 문단 길이 | 50~80자 |
| 아웃트로 | 구독 CTA 포함 |

> **팁:** 대본 스킬은 "뉴스 대본 써줘"처럼 키워드가 포함되면 자동 로드됩니다. "대본 써줘"만 말하면 로드되지 않으므로 활성화 키워드를 잘 설정하세요.

> **핵심:** 대본 템플릿 스킬 = 포맷을 매번 설명하지 않아도 키워드 하나로 정해진 구조의 대본이 나옵니다.

## Part 6: 실습 3 -- 콘텐츠 시리즈 스킬

**목표:** 시리즈물의 일관된 구조를 스킬로 정의

### Step 1: 스킬 정의

```
/skill-creator

"'AI 도구 리뷰' 시리즈 스킬을 만들어줘.

규칙:
1. 대본 구조: 도구 소개(2문단) → 핵심 기능 3개(각 2문단) → 장단점(2문단) → 추천 대상(1문단) → 아웃트로(1문단)
2. 인트로: 항상 '오늘 소개할 도구는 [도구명]입니다' 패턴
3. 슬라이드: 첫 슬라이드에 도구 로고/아이콘 큰 배치
4. 장단점 슬라이드: 항상 좌우 비교 레이아웃 사용
5. 추천 슬라이드: 타겟 사용자 아이콘 3개
6. 시리즈 넘버링: 제목에 '#N' 포함

활성화 조건: '도구 리뷰', 'AI 도구', '리뷰 영상' 키워드 시"
```

### Step 2: 생성 결과 확인

```
.claude/skills/ai-tool-review/
├── SKILL.md                    ← 인덱스 + 대본 구조
├── chapter-slide-rules.md      ← 슬라이드 레이아웃 규칙
└── chapter-series-format.md    ← 시리즈 넘버링, 연결 패턴
```

### Step 3: 테스트

```
"Cursor AI 리뷰 영상 만들어줘"
→ 시리즈 패턴 자동 적용
```

테스트 체크리스트:

| 확인 항목 | 기대값 |
|----------|--------|
| 인트로 패턴 | "오늘 소개할 도구는 Cursor AI입니다" |
| 대본 구조 | 소개 → 기능 3개 → 장단점 → 추천 → 아웃트로 |
| 첫 슬라이드 | 도구 아이콘/로고 큰 배치 |
| 장단점 슬라이드 | 좌우 비교 레이아웃 |
| 제목 넘버링 | "#N" 포함 |

> **팁:** 시리즈 스킬은 대본 + 슬라이드 규칙이 함께 들어갑니다. 400줄을 넘기 쉬우므로 chapter 파일로 분리되는 것이 정상입니다.

> **핵심:** 시리즈 스킬 = 대본 구조 + 슬라이드 레이아웃 + 넘버링을 하나로 묶어 시리즈 일관성을 자동 관리.

## Part 7: 스킬 관리와 개선

### 스킬 수정하기

기존 스킬에 규칙을 추가하거나 변경할 때도 `/skill-creator` (Claude Code 내장 스킬)를 사용합니다:

```
/skill-creator
"기존 TechBrief 스타일 스킬에 '글로우 효과 추가' 규칙을 넣어줘"
```

```
/skill-creator
"뉴스 대본 스킬에서 '포인트 3개'를 '포인트 4~5개'로 변경해줘"
```

> 스킬 파일을 직접 편집하지 마세요. `/skill-creator` (Claude Code 내장 스킬)가 구조를 유지하면서 수정합니다.

### 스킬 파일 구조 이해

```
.claude/skills/my-style/
├── SKILL.md                  ← 인덱스 (400줄 이하)
├── chapter-color-rules.md    ← 색상 규칙
├── chapter-layout-rules.md   ← 레이아웃 규칙
└── chapter-animation.md      ← 애니메이션 규칙
```

| 파일 | 역할 | 로드 시점 |
|------|------|----------|
| `SKILL.md` | 인덱스 + 활성화 조건 + 핵심 규칙 | 항상 로드 |
| `chapter-*.md` | 세부 규칙 | 필요할 때만 선택적 로드 |

> 스킬이 400줄 넘으면 chapter 파일로 분리됩니다. Claude Code가 필요한 부분만 선택적으로 로드하므로 성능에 영향을 주지 않습니다.

### autoresearch로 스킬 최적화 (고급)

스킬의 결과물 품질을 자동으로 개선하고 싶다면 autoresearch를 활용합니다:

```
"TechBrief 스타일 스킬을 autoresearch로 최적화해줘.
슬라이드가 더 일관된 결과를 내도록 개선해줘."
```

autoresearch의 작동 방식:

```
스킬 실행 → 결과 평가 (이진: 성공/실패) → 프롬프트 변형 → 재실행
    ↑                                                      │
    └──────────── 개선된 스킬 저장 ←─────────────────────────┘
```

- 이진 평가(성공/실패) 기반으로 프롬프트를 자동 변형
- 여러 번 반복 실행하며 가장 일관된 결과를 내는 버전을 채택
- 결과 로그와 변경 이력이 자동 저장

> **핵심:** 스킬 수정은 항상 `/skill-creator` (Claude Code 내장 스킬)로. 400줄 초과 시 chapter 분리. autoresearch 스킬로 품질 자동 최적화 가능.

## Part 8: 실전 사례 — 슬라이드 스킬을 개선한 이야기

스킬은 한번 만들면 끝이 아닙니다. 실제로 이 시스템의 슬라이드 생성 스킬을 두 달에 걸쳐 개선한 경험을 공유합니다.

### 처음: "규칙을 많이 넣으면 좋아지겠지"

슬라이드 생성 스킬을 처음 만들 때, 결과물의 품질을 높이려고 규칙을 잔뜩 넣었습니다:

```
- "영상의 1구간은 보라색, 2구간은 청록색, 3구간은 코랄색을 써라"
- "레이아웃 다양성 점수를 4개 항목으로 수치화해서 관리해라"
- "각 구간마다 사용할 색상 조합을 사전에 정의해둬라"
```

의도는 좋았습니다. "구체적으로 지시하면 결과가 좋아지겠지"라는 생각이었죠.

### 결과: 오히려 단조로워짐

그런데 실제 결과물을 보니 **레이아웃이 기계적이고 단조로웠습니다**. 같은 패턴의 슬라이드가 반복되고, 색상도 딱딱하게 구간별로 잘려서 자연스럽지 않았습니다.

규칙이 너무 많아서 Claude가 상황에 맞는 창의적 판단을 할 여지가 없었던 겁니다.

### 진단: 같은 대본으로 비교해보기

개선 전에 한 가지 중요한 준비를 했습니다. **기존 결과물을 백업**해둔 겁니다:

```
slides-old/   ← 기존 스킬로 만든 슬라이드 (백업)
slides/       ← 새 스킬로 다시 만들 슬라이드
```

같은 대본으로 다시 만들어서 나란히 비교하면, "좋아졌다"가 느낌이 아니라 눈에 보이는 증거가 됩니다.

### 개선: 규칙을 빼고 원칙을 넣다

스킬 업데이트의 핵심은 **규칙 추가가 아니라 불필요한 규칙 제거**였습니다:

| 빼낸 것 (세부 규칙) | 넣은 것 (원칙) |
|----------|------|
| "1구간은 보라색 써라" | "핵심 개념은 강조색, 해결책은 청록색" |
| "다양성을 수치로 관리해라" | "같은 레이아웃 3연속 금지, 단순하면 한 단계 더 풍부하게" |
| "색상 조합을 사전에 정의" | "대본의 감정 흐름에 맞게 자유롭게 변화" |

**차이를 한마디로:** "1번 문단은 #7C7FD9를 써라"(구체적 지시) → "핵심 개념에는 강조색을 써라"(원칙). 원칙은 Claude에게 판단의 여지를 주고, 세부 지시는 기계적 순응만 유도합니다.

### 결과

같은 대본으로 슬라이드를 전부 다시 만들었습니다:

```
Before (세부 규칙 많았을 때):
- 비슷한 레이아웃이 계속 반복 (비교형이 전체의 40%)
- 색상이 구간별로 뚝뚝 끊김
- "AI가 만든 느낌"이 강함

After (원칙 기반으로 바꾼 후):
- 20가지 이상의 레이아웃이 자연스럽게 섞임
- 대본 흐름에 따라 색상이 부드럽게 변화
- 애니메이션 속도도 다양해져서 편집감이 살아남
```

### 여러분이 가져갈 수 있는 교훈 4가지

**1. 이전 결과물을 항상 백업하세요**

스킬을 수정하기 전에 기존 결과물을 따로 저장해두세요. 같은 입력으로 다시 만들어 비교하면 개선 여부가 명확해집니다.

**2. 규칙을 빼는 것도 개선입니다**

스킬에 규칙을 추가하는 것만이 개선이 아닙니다. "이 규칙이 아직 필요한가?"를 정기적으로 확인하세요. 불필요한 규칙을 걷어내면 오히려 품질이 올라갈 수 있습니다.

**3. 세부 지시보다 원칙이 낫습니다**

"빨간색 써라"보다 "경고 상황에는 눈에 띄는 색을 써라"가 더 좋은 결과를 냅니다. Claude에게 **왜** 그렇게 해야 하는지를 알려주면, 상황에 맞게 스스로 판단합니다.

**4. 이 사이클을 반복하면 스킬이 계속 진화합니다**

```
기존 결과물 백업 → 스킬 수정 → 같은 대본으로 재생성 → 비교 → 반복
```

이 과정을 2~3회만 반복해도 스킬의 품질이 눈에 띄게 달라집니다. 자동화하고 싶다면 autoresearch 스킬도 활용할 수 있습니다.

> **핵심:** 스킬은 한번 만들고 끝이 아닙니다. "이전 결과물 백업 → 스킬 수정 → 비교"를 반복하면 스킬이 진짜 자산으로 성장합니다. 특히 "규칙을 빼는 것도 개선"이라는 점을 기억하세요.

## Part 9: 스킬 아이디어 모음 + 도메인별 프롬프트

어떤 스킬을 만들지 고민된다면 아래 표에서 영감을 얻으세요:

| 스킬 이름 | 용도 | 활성화 조건 |
|----------|------|------------|
| 채널 스타일 | 내 채널 디자인 규칙 (색상, 폰트, 레이아웃) | 슬라이드 생성 시 |
| 뉴스 대본 | 3분 뉴스 요약 템플릿 | "뉴스 대본", "속보" |
| 시리즈 패턴 | 시리즈물 구조 통일 (넘버링, 연결) | "리뷰", "시리즈" |
| 썸네일 스타일 | 썸네일 디자인 규칙 (텍스트 배치, 색상) | "썸네일" |
| 인트로 패턴 | 후킹 인트로 3종 랜덤 적용 | "인트로", "대본 시작" |
| 번역 대본 | 영어 원문 → 한국어 로컬라이징 규칙 | "번역", "영문 대본" |
| 쇼츠 컷 | 쇼츠 전용 짧은 대본 (30~60초) | "쇼츠 대본", "바이럴" |

### 도메인별 스킬 생성 프롬프트 (집에서 복사해서 사용)

**재테크 전용 대본 스킬:**
```
/skill-creator

"'재테크 분석' 대본 템플릿 스킬을 만들어줘.

규칙:
1. 인트로: 시청자 공감 질문형 ("~하고 계시죠?")
2. 본문: 분석 항목 3~5개, 각 항목마다 (현황→원인→대응) 구조
3. 모든 포인트에 구체적 숫자/금액/수익률 포함 필수
4. 아웃트로: "투자는 개인 판단" 면책 + 구독 CTA
5. 문단당 60~80자, 총 10~12 문단
6. 슬라이드: 숫자 강조(80px+), 바차트/비교표 레이아웃 선호

활성화 조건: '재테크 대본', 'ETF', '주식 분석', '투자' 키워드 시"
```

**운동/루틴 전용 대본 스킬:**
```
/skill-creator

"'운동 루틴' 대본 템플릿 스킬을 만들어줘.

규칙:
1. 인트로: 공감형 ("~하고 계시죠? 오늘 N가지 동작으로 해결해 드릴게요.")
2. 본문: 동작 N개, 각 동작 = 1문단 (동작명 + 자세 + 시간/횟수)
3. 동작 간 전환 문장: "다음 동작으로 넘어갑니다" 패턴
4. 아웃트로: "매일 하면 N주 후 변화" + 구독 CTA
5. 슬라이드: 동작 번호 크게(80px), 스텝 표시기 레이아웃

활성화 조건: '운동 대본', '루틴', '스트레칭', '홈트' 키워드 시"
```

**요리 레시피 전용 대본 스킬:**
```
/skill-creator

"'레시피' 대본 템플릿 스킬을 만들어줘.

규칙:
1. 인트로: "오늘은 N분 만에 만드는 [요리명]입니다" 패턴
2. 재료 문단: 재료 목록을 자연스러운 문장으로 (분량 포함)
3. 조리 과정: 단계별 1문단씩 (1. 재료 손질 → 2. 조리 → 3. 플레이팅)
4. 아웃트로: "이 레시피 저장하고 싶으시면" + 구독 CTA
5. 슬라이드: 재료 문단은 체크리스트, 조리 과정은 스텝 번호 레이아웃

활성화 조건: '레시피 대본', '요리', '자취 요리' 키워드 시"
```

**어학/교육 전용 대본 스킬:**
```
/skill-creator

"'영어 강의' 대본 템플릿 스킬을 만들어줘.

규칙:
1. 인트로: 학습자 Pain Point 질문 ("~할 때 어떻게 말하시나요?")
2. 본문: 핵심 표현/문법 3~5개, 각각 (잘못된 예 → 올바른 예 → 활용 상황)
3. 예문은 항상 영어 원문 + 한국어 해석 함께
4. 아웃트로: 오늘 배운 표현 요약 + "매일 1개씩" 동기부여 + 구독 CTA
5. 슬라이드: Before/After 비교 레이아웃, 핵심 표현은 72px Bold

활성화 조건: '영어 대본', '회화', '영어 강의', '어학' 키워드 시"
```

### 조합 패턴

스킬은 단독으로 쓸 수도 있지만, 조합하면 더 강력합니다:

```
채널 스타일 스킬 + 뉴스 대본 스킬
= "뉴스 대본 써줘"만으로 → 내 채널 스타일의 뉴스 대본 + 슬라이드 자동 생성

시리즈 패턴 스킬 + 인트로 패턴 스킬
= 시리즈 구조를 유지하면서 매 에피소드 다른 훅으로 시작
```

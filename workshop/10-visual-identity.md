# 비주얼 아이덴티티 구축

> 나만의 채널 디자인 시스템 만들기

> 난이도: 심화 | 소요: 읽기 15분 + 실습 30분

---

## Part 1: 비주얼 아이덴티티란?

"이 영상 어디서 봤지?" -- 이런 반응이 나온다면 이미 브랜딩에 성공한 겁니다. 시청자가 썸네일이나 첫 3초만 보고도 채널을 알아보는 것, 이것이 **비주얼 아이덴티티**입니다.

비주얼 아이덴티티를 구성하는 네 가지 축:

```
색상 팔레트 + 폰트 체계 + 레이아웃 패턴 + 캐릭터 스타일
= 채널의 시각적 얼굴
```

일관된 스타일은 **신뢰감**을 만들고, 신뢰감은 **구독 전환**으로 이어집니다. 이 시스템에서는 Skill + Config 조합으로 한번 정의하면 이후 모든 영상에 자동 적용할 수 있습니다.

> **핵심:** 비주얼 아이덴티티 = 시청자가 내 채널을 "알아보게" 만드는 시각 시스템. Skill로 자산화하면 매번 지정할 필요 없음.

## Part 2: 4가지 구성요소

### 2-1. 색상 팔레트

영상의 분위기를 결정하는 가장 강력한 요소입니다. 3~4색만 정하면 충분합니다.

| 요소 | 역할 | 설정 위치 | 예시 |
|------|------|----------|------|
| 배경색 | 전체 분위기 | 슬라이드 TSX | `#0a0a1a` (다크) |
| 포인트색 | 강조/하이라이트 | 슬라이드 TSX | `#00d4ff` (시안) |
| 텍스트색 | 본문 가독성 | 슬라이드 TSX / 자막설정 | `#ffffff` (흰색) |
| 서브텍스트색 | 보조 정보 | 슬라이드 TSX | `#888888` (회색) |

> **팁:** 포인트색은 배경과 명도 차이가 클수록 좋습니다. 다크 배경에는 밝은 포인트, 밝은 배경에는 진한 포인트.

### 2-2. 타이포그래피

폰트와 크기 체계를 미리 정해두면 슬라이드마다 고민할 필요가 없습니다.

| 용도 | 추천 폰트 | Weight | 크기 |
|------|----------|--------|------|
| 제목 | Pretendard | Bold / Black | 64~80px |
| 부제 | Pretendard | SemiBold | 40~48px |
| 본문 | Pretendard | Regular / Medium | 32~36px |

Pretendard는 한국어와 영문 모두 깔끔하게 표현되어 기본 선택으로 적합합니다. 채널 성격에 따라 Noto Sans KR, Wanted Sans 등을 선택해도 좋습니다.

### 2-3. 레이아웃 패턴

자주 쓰는 3~4가지 레이아웃을 미리 정해두면 일관성이 올라갑니다. 매번 새로운 레이아웃을 쓰면 채널 아이덴티티가 흐려집니다.

| 전략 | 예시 |
|------|------|
| 미니멀리스트 | "내 채널은 큰제목형, 리스트형, 비교형만 쓴다" |
| 다이어그램 중심 | "플로차트, 타임라인, 바차트를 주로 쓴다" |
| UI 모형 중심 | "앱 화면, 채팅 UI, 대시보드를 자주 쓴다" |

> 05-slide-generation.md에서 레이아웃 종류를 확인할 수 있습니다.

### 2-4. B-roll 캐릭터

B-roll에 등장하는 캐릭터가 채널의 마스코트 역할을 합니다. `config.api.yaml`에서 한번 설정하면 모든 B-roll에 일관되게 적용됩니다.

```yaml
# config/config.api.yaml
broll:
  character_description: "cute male chibi anime character with dark brown hair, wearing a hoodie, minimal flat illustration style"
```

변경 예시:

```yaml
# 비즈니스 채널
character_description: "professional female business character, clean flat illustration style, navy suit"

# 게임 채널
character_description: "pixel art robot character, retro 8-bit style, green neon glow"
```

> **핵심:** 캐릭터 설명을 바꾸는 것만으로 채널 전체의 B-roll 분위기가 달라집니다.

## Part 3: 채널 유형별 추천 아이덴티티

### 기술/IT 채널

| 요소 | 추천 |
|------|------|
| 배경 | `#0a0a1a` ~ `#0f1923` (다크 네이비) |
| 포인트 | `#00d4ff` (시안) 또는 `#00ff88` (그린) |
| 분위기 | 미래적, 미니멀, 테크 |
| TTS 속도 | 1.1~1.15 (빠른 정보전달) |
| 캐릭터 | 로봇/사이버 느낌 |
| 레이아웃 | 다이어그램, 코드 블록, 플로차트 |

### 교육/튜토리얼 채널

| 요소 | 추천 |
|------|------|
| 배경 | `#f5f5f5` ~ `#ffffff` (밝은 톤) |
| 포인트 | `#3b82f6` (블루) |
| 분위기 | 깔끔, 신뢰, 교과서적 |
| TTS 속도 | 0.95~1.0 (차분하게) |
| 캐릭터 | 친근한 선생님/가이드 |
| 레이아웃 | 스텝 표시기, 번호 리스트, 비교표 |

### 뉴스/트렌드 채널

| 요소 | 추천 |
|------|------|
| 배경 | `#0f0f0f` (블랙) |
| 포인트 | `#ff4444` (레드) 또는 `#ff8800` (오렌지) |
| 분위기 | 긴급, 임팩트, 속보 |
| TTS 속도 | 1.1~1.15 |
| 캐릭터 | 뉴스 앵커/리포터 느낌 |
| 레이아웃 | 큰 제목, 인용문, 타임라인 |

### 라이프스타일 채널

| 요소 | 추천 |
|------|------|
| 배경 | `#fef3e2` (크림) |
| 포인트 | `#e8a87c` (코랄/살몬) |
| 분위기 | 따뜻, 부드러운, 감성적 |
| TTS 속도 | 0.9~1.0 |
| 캐릭터 | 수채화/파스텔 일러스트 |
| 레이아웃 | 큰 이미지, 인용문, 심플 텍스트 |

> **팁:** 위 표는 출발점일 뿐입니다. 내 채널의 시청자를 떠올리며 조합을 조정하세요.

## Part 4: 설정 파일로 적용하기

### 바꿀 수 있는 설정 위치

```yaml
# config/config.api.yaml

# TTS 음성 스타일
tts:
  elevenlabs:
    speed: 1.15           # 채널 톤에 맞게 (0.9~1.2)
    stability: 0.85        # 안정적(1.0) vs 감정적(0.3)
    similarity_boost: 0.58 # 원본 목소리 유사도

# B-roll 캐릭터
broll:
  character_description: "your character description here"

# 자막 스타일
subtitles:
  font_size: 20
  color: "white"
  background: "rgba(0,0,0,0.6)"
```

> 09-config-customization.md 참조 -- 각 설정의 세부 옵션이 자세히 설명되어 있습니다.

### Claude Code 프롬프트 예시

설정을 직접 편집하지 않아도 Claude Code에게 자연어로 요청할 수 있습니다:

```
TTS 속도를 1.1로 변경해줘
```

```
B-roll 캐릭터를 "professional robot character, clean minimal style"로 바꿔줘
```

```
자막 배경을 투명하게 해줘
```

```
포인트 색상을 시안(#00d4ff)에서 오렌지(#ff8800)로 바꿔줘
```

## Part 5: Skill로 자산화하기

설정을 일일이 기억하고 매번 지정하는 건 비효율적입니다. 채널 스타일을 Skill로 만들어두면 `/generate-video` 실행 시 자동 적용됩니다.

### 과정

```
[1] 리버스 프롬프팅으로 내 스타일 패턴 추출
    (05-slide-generation.md 참조)
    ↓
[2] 색상/폰트/레이아웃/캐릭터 정리
    ↓
[3] /skill-creator (Claude Code 내장 스킬)로 스킬 생성
    ↓
[4] 이후 영상 생성 시 자동 적용
```

### 도메인별 스킬 생성 프롬프트 (골라서 사용하세요)

**IT/테크 뉴스 채널:**
```
/skill-creator

"내 채널 'TechBrief'의 스타일 스킬을 만들어줘:
- 배경: 어두운 네이비 (#0f1923)
- 포인트: 시안 (#00d4ff)
- 슬라이드: 미니멀, 텍스트 중심, 다이어그램 선호
- 폰트: 제목 Bold 72px, 본문 Regular 36px
- 애니메이션: 빠른 페이드인 (0.3초)
- 금지: 파스텔톤, 둥근 모서리, 귀여운 캐릭터"
```

**재테크/투자 채널:**
```
/skill-creator

"내 채널 '머니브리핑'의 스타일 스킬을 만들어줘:
- 배경: 짙은 네이비 (#0b1426)
- 포인트: 골드 (#d4a726), 서브: 민트 (#4ade80)
- 숫자/금액은 항상 80px 이상 Bold로 크게 강조
- 차트(바, 파이), 비교표 레이아웃 선호
- 금지: 귀여운 일러스트, 라운드 카드, 파스텔"
```

**교육/튜토리얼 채널:**
```
/skill-creator

"내 채널 '쉬운클래스'의 스타일 스킬을 만들어줘:
- 배경: 밝은 크림 (#faf8f0)
- 포인트: 딥블루 (#2563eb), 서브: 오렌지 (#f97316)
- 단계별 번호 리스트(1→2→3) 선호
- 핵심 용어에 하이라이트(배경 #2563eb, opacity 0.15)
- 폰트: 제목 SemiBold 64px, 본문 Regular 34px
- 금지: 다크 배경, 네온 색상"
```

**건강/피트니스 채널:**
```
/skill-creator

"내 채널 '데일리핏'의 스타일 스킬을 만들어줘:
- 배경: 다크 그린 (#0a1a0a)
- 포인트: 라임 (#88ff00), 서브: 화이트 (#ffffff)
- 스텝 번호를 크게(80px), 동작 설명은 아이콘+텍스트 조합
- 에너지 넘치는 spring 애니메이션 (mass:1, damping:10)
- 금지: 파스텔, 정적인 텍스트만 있는 레이아웃"
```

**요리/라이프스타일 채널:**
```
/skill-creator

"내 채널 '오늘의레시피'의 스타일 스킬을 만들어줘:
- 배경: 따뜻한 크림 (#fef3e2)
- 포인트: 코랄 (#ff6b6b), 서브: 브라운 (#8b5e3c)
- 레시피 카드 스타일: 부드러운 그림자, radius 20px
- 재료/분량은 별도 박스로 분리
- 폰트: 제목 SemiBold 60px, 본문 Regular 32px
- 금지: 다크 테마, 기술적 다이어그램, 네온"
```

**여행 채널:**
```
/skill-creator

"내 채널 '트래블로그'의 스타일 스킬을 만들어줘:
- 배경: 따뜻한 베이지 (#f5ede0)
- 포인트: 코랄 (#ff6b6b), 서브: 스카이블루 (#38bdf8)
- 큰 배경 이미지 위에 반투명 텍스트 박스 오버레이 선호
- 일정/코스는 타임라인 레이아웃 사용
- 금지: 다크 테마, 차트/그래프, 코드 블록"
```

스킬이 저장되면 이후 `/generate-video`를 실행할 때 이 스타일이 자동으로 반영됩니다. 직접 색상이나 폰트를 지정할 필요가 없어집니다.

> **핵심:** 반복되는 스타일 지시 = Skill로 자산화. 한번 만들면 계속 쓸 수 있음.

아래 워크시트를 채워보세요. 완성 후 `/skill-creator` (Claude Code 내장 스킬)로 스킬을 만들면 됩니다.

```
  배경: #____________
  포인트: #____________
  텍스트: #____________
  서브텍스트: #____________

분위기 키워드 3가지: ___, ___, ___

폰트:
  제목: ___________ (weight: ___, size: ___px)
  본문: ___________ (weight: ___, size: ___px)

TTS 속도: ___
TTS stability: ___

캐릭터 설명:
_______________________________________________

1. _______________

3. _______________

```

### 실습 순서

1. 워크시트를 채운다
2. Part 3의 채널 유형별 추천을 참고하되 그대로 복사하지 않는다
3. Claude Code에서 `/skill-creator` (Claude Code 내장 스킬)를 실행하고 워크시트 내용을 전달한다
4. 생성된 스킬로 테스트 영상을 만들어 스타일을 확인한다

```
/skill-creator

"아래 워크시트를 기반으로 내 채널 스타일 스킬을 만들어줘:
[워크시트 내용 붙여넣기]"
```

> **핵심:** 워크시트를 채운 뒤 `/skill-creator` (Claude Code 내장 스킬)로 스킬을 만들어보세요. 정답은 없습니다 -- 내 채널에 맞는 스타일이 정답입니다.

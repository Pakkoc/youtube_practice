# 품질 반복 개선 가이드

> 결과물을 체계적으로 개선하는 방법

> 난이도: 중급 | 소요: 읽기 15분 + 실습 20분

---

## Part 1: 첫 결과물은 시작일 뿐

자동 생성은 **빠른 초안 제작** 도구입니다. 좋은 영상은 초안에서 나오는 게 아니라 **반복 개선(iteration)**에서 나옵니다.

```
"한번에 완벽하게" (X)
"빠르게 만들고 개선하기" (O)
```

첫 생성에서 80점짜리가 나왔다면, 2~3회 반복으로 95점까지 끌어올릴 수 있습니다. 처음부터 100점을 노리면 오히려 느려집니다.

> **핵심:** 첫 결과물을 판단 기준으로 쓰지 마세요. 반복 개선이 품질을 결정합니다.

## Part 2: 5-Point 리뷰 체크리스트

영상을 처음부터 끝까지 한번 재생한 뒤, 아래 체크리스트로 평가합니다.

| 영역 | 체크포인트 | 평가 (O/X) |
|------|-----------|:----------:|
| **음성** | 발음이 자연스러운가? | |
| | 속도가 적절한가? | |
| | 영어 단어 발음이 맞는가? | |
| **슬라이드** | 텍스트가 읽기 쉬운가? | |
| | 레이아웃이 다양한가? (같은 패턴 반복 X) | |
| | 색상이 일관되는가? | |
| **B-roll** | 내용과 맥락이 맞는가? | |
| | 스타일이 통일되어 있는가? | |
| **자막** | 타이밍이 음성과 맞는가? | |
| | 글자가 충분히 큰가? | |
| **전체 흐름** | 인트로가 시선을 끄는가? | |
| | 문단 간 전환이 자연스러운가? | |
| | 아웃트로가 행동을 유도하는가? | |

X가 2개 이상인 영역이 가장 먼저 개선할 대상입니다. 한번에 모든 걸 고치려 하지 말고, **가장 큰 문제 1~2개**에 집중하세요.

> **팁:** 이 체크리스트를 프로젝트 폴더에 `review.md`로 저장해두면 매번 꺼내 쓸 수 있습니다.

## Part 3: 문제별 해결 가이드

### 음성 관련

| 증상 | 원인 | 해결 |
|------|------|------|
| "딱딱하게 읽어요" | stability 값이 높음 | `stability: 0.5~0.7`로 낮추기 |
| "너무 빨라요" | speed 높음 | `speed: 0.95~1.0`으로 조정 |
| "영어를 이상하게 읽어요" | TTS 사전 미등록 | `발음 사전에 "단어=발음" 추가해줘` |
| "감정이 없어요" | stability 높음 + style 낮음 | stability 낮추기 |
| "끊겨 들려요" | 문장이 너무 길음 | 대본에서 문장 분리 |

Claude Code 프롬프트:

```
TTS stability를 0.6으로 낮춰줘
```

```
발음 사전에 "Claude=클로드", "GPT=지피티" 추가해줘
```

> 09-config-customization.md 참조 -- TTS 설정 옵션 상세 설명

### 슬라이드 관련

| 증상 | 원인 | 해결 |
|------|------|------|
| "다 비슷해 보여요" | 레이아웃 반복 | `레이아웃 다양하게 바꿔줘` |
| "글자가 안 보여요" | 배경과 텍스트 대비 부족 | 배경 어둡게 or 텍스트에 그림자 |
| "너무 복잡해요" | 한 슬라이드에 정보 과다 | 문단 분리 (대본 수정) |
| "촌스러워요" | 색상 불일치 | 10-visual-identity.md로 팔레트 통일 |
| "애니메이션이 없어요" | TSX에 spring 미적용 | `애니메이션 추가해줘` |

Claude Code 프롬프트:

```
3번 슬라이드 레이아웃을 비교형으로 바꿔줘
```

```
전체 슬라이드에 페이드인 애니메이션 넣어줘
```

> 05-slide-generation.md 참조 -- 레이아웃 패턴 목록

### B-roll 관련

| 증상 | 원인 | 해결 |
|------|------|------|
| "맥락이 안 맞아요" | B-roll 프롬프트 부정확 | `B-roll 프롬프트를 수정해줘` |
| "스타일이 들쭉날쭉" | character_description 미설정 | 캐릭터 설명 통일 (10-visual-identity.md 참조) |
| "너무 단순해요" | 프롬프트에 디테일 부족 | 구체적인 장면 묘사 추가 |

Claude Code 프롬프트:

```
2번 문단 B-roll 프롬프트를 "서버실에서 데이터가 흘러가는 장면"으로 바꿔줘
```

> **핵심:** 증상을 정확히 파악하면 해결이 빠릅니다. "별로다"가 아니라 "왜 별로인지"를 찾으세요.

## Part 4: 설정 튜닝 기록 유지

설정을 바꿨는데 결과가 좋았다면, 기록을 남겨야 다음에도 쓸 수 있습니다. 기록하지 않으면 같은 삽질을 반복하게 됩니다.

### 기록 템플릿

```
[ 설정 튜닝 기록 ]
날짜: 2026-03-26
프로젝트: my-project-003
변경 항목: TTS speed
변경 전: 1.15
변경 후: 1.0
결과: 더 자연스럽게 들림 -- 채택
```

### Claude Code 메모리 활용

변경 결과가 좋으면 Claude Code에게 기억시킬 수 있습니다:

```
"TTS speed 1.0이 내 채널에 잘 맞아. 기억해줘."
```

```
"B-roll 캐릭터 설명을 'minimal flat robot'으로 바꿨더니 좋았어. 기억해줘."
```

Claude Code의 메모리에 저장되면 다음 대화에서도 이 설정을 기본으로 제안합니다.

> **팁:** 좋은 설정을 발견하면 기록 + 메모리 저장 + Skill 반영, 세 곳에 모두 남기는 것이 가장 확실합니다.

## Part 5: 부분 재생성으로 빠르게 반복

전체 파이프라인을 처음부터 재실행할 필요가 없습니다. 문제가 있는 부분만 골라서 다시 만들면 됩니다.

### 반복 개선 사이클

```
[1] 문제 파악 (리뷰 체크리스트)
    |
    v
[2] 설정 또는 슬라이드 수정
    |
    v
[3] 부분 재생성
    |
    v
[4] 결과 확인 --> 만족? --> 완료
         |
         v (불만족)
    [2]로 돌아가기
```

### 부분별 재생성 명령

| 수정 대상 | 재생성 방법 |
|----------|-----------|
| TTS (발음/속도) | `uv run python scripts/continue_pipeline.py <project>` (TTS 단계부터 재실행) |
| 슬라이드 (레이아웃/색상) | `uv run python scripts/regenerate_slides.py <project>` |
| B-roll (이미지) | `uv run python scripts/regenerate_broll.py <project>` |
| 자막 (타이밍/크기) | `uv run python scripts/recompose_video.py <project>` (자막 포함 재합성) |
| 최종 합성 | `uv run python scripts/recompose_video.py <project>` |

### 일반적인 수정 흐름

```
슬라이드만 수정한 경우:
  regenerate_slides.py --> recompose_video.py

TTS만 수정한 경우:
  continue_pipeline.py (TTS 단계부터 재실행) --> recompose_video.py

둘 다 수정한 경우:
  continue_pipeline.py (TTS 단계부터 재실행) --> regenerate_slides.py --> recompose_video.py
```

> **핵심:** 전체 재실행 대신 부분 재생성을 쓰면 수 분 안에 결과를 확인할 수 있습니다.

## Part 6: Skill 개선 사이클

반복 개선 중에 같은 문제가 계속 나타나면, 그것은 Skill을 업데이트할 타이밍입니다.

### 패턴 발견 -> Skill 업데이트

| 반복되는 문제 | Skill에 추가할 규칙 |
|-------------|-------------------|
| "매번 레이아웃이 단조롭다" | slide-generation 스킬에 다양성 규칙 추가 |
| "B-roll이 항상 맥락을 벗어난다" | broll 스킬에 맥락 매칭 가이드라인 추가 |
| "색상이 자꾸 달라진다" | 스타일 스킬에 색상 팔레트 고정 규칙 추가 |

### Skill 업데이트 방법

```
/skill-creator

"slide-generation 스킬을 업데이트해줘:
- 연속 3개 이상 같은 레이아웃 패턴 금지
- 비교형, 리스트형, 다이어그램 중 최소 2가지 포함"
```

기존 스킬을 수정하는 것이므로 새로 만들 필요 없이 `/skill-creator`로 업데이트하면 됩니다.

### autoresearch로 자동 최적화

수동 개선을 넘어서, autoresearch 스킬로 자동 최적화할 수도 있습니다. 이 부분은 고급 주제이므로 별도로 다룹니다.

### 스킬 진화의 실전 사례

실제로 이 시스템의 슬라이드 생성 스킬을 개선한 사례가 있습니다. 처음엔 "색상은 이걸 써라, 다양성 점수를 수치로 관리해라" 같은 세부 규칙을 잔뜩 넣었는데, 오히려 결과가 기계적이었습니다. 세부 규칙을 빼고 "핵심 개념은 강조색을 써라, 같은 레이아웃 3연속 금지" 같은 원칙으로 바꿨더니 품질이 확 올라갔습니다.

**규칙을 추가하는 것만이 개선이 아니라, 불필요한 규칙을 빼는 것도 개선**이라는 것이 핵심 교훈입니다.

> 상세 사례: `13-custom-skills.md` Part 8 "실전 사례 — 슬라이드 스킬을 개선한 이야기" 참조

> **핵심:** 같은 문제가 3번 이상 반복되면 Skill에 규칙을 추가하세요. 반대로 결과가 기계적으로 느껴지면 불필요한 규칙을 빼보세요. 이후 영상에서는 자동으로 반영됩니다.

## Part 7: 실습 -- 첫 영상 리뷰 & 개선

이전에 만든 영상이 있다면, 지금 바로 개선해봅시다.

### 단계별 진행

**1단계: 리뷰**

영상을 처음부터 끝까지 재생하면서 Part 2의 5-Point 체크리스트를 작성합니다.

```
[ 리뷰 결과 ]
음성: O / O / X (영어 발음)
슬라이드: O / X (레이아웃 반복) / O
B-roll: X (맥락 불일치) / O
자막: O / O
전체 흐름: O / O / X (아웃트로 약함)
```

**2단계: 우선순위 선정**

X 중에서 가장 큰 문제 1개를 선택합니다. 여러 개를 한번에 고치려 하지 마세요.

```
가장 큰 문제: 슬라이드 레이아웃 반복
```

**3단계: 수정**

Claude Code에게 구체적으로 요청합니다:

```
my-project의 슬라이드 레이아웃을 다양하게 바꿔줘.
2번, 4번, 6번이 다 비슷한 큰제목형인데, 4번을 비교표, 6번을 플로차트로 바꿔줘.
```

**품질 개선 프롬프트 모음 (증상별로 골라 쓰세요):**

```
전체 슬라이드의 텍스트 대비를 높여줘. 배경이 밝은 곳은 어두운 텍스트, 배경이 어두운 곳은 흰색+텍스트 그림자.
```

```
슬라이드 간 시각 변화가 너무 없어. 3, 5, 7번 슬라이드에 다른 배경색 변형을 줘서 리듬감 추가해줘.
```

```
인트로(1번) 슬라이드가 밋밋해. 제목을 96px로 키우고, 배경에 미세한 방사형 그라데이션 추가해줘.
```

```
아웃트로(마지막) 슬라이드에 구독 유도 CTA를 시각적으로 강조해줘. 화살표 아이콘 + 강조색 박스.
```

```
B-roll 이미지가 대본 맥락과 안 맞아. 3번 문단의 B-roll 프롬프트를 "팀원들이 화면을 보며 논의하는 장면"으로 구체적으로 바꿔줘.
```

```
발음 사전을 대본 기준으로 자동 보강해줘. 영어 단어, 약어, 기술 용어를 자동으로 찾아서 한글 발음 등록.
```

**4단계: 부분 재생성**

```bash
# 슬라이드 재생성
uv run python scripts/regenerate_slides.py my-project

# 최종 합성
uv run python scripts/recompose_video.py my-project
```

**5단계: 비교**

수정 전/후 영상을 비교합니다. 개선이 확인되면 다음 문제로 넘어갑니다.

> **핵심:** 리뷰 -> 1개 선택 -> 수정 -> 확인. 이 사이클을 2~3회 반복하면 눈에 띄게 달라집니다.
